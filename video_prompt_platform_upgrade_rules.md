# 영상 프롬프트 플랫폼별 업그레이드 규칙

이 문서는 《棺中檀色》/《黑棺里的女人》 영상 생성 작업에서 누적된 실패 경험을 바탕으로, **即梦 / Seedance 2.0**과 **Liblib / LibTV**를 번갈아 사용할 때 프롬프트 출력 형식을 혼동하지 않기 위한 공용 규칙이다.

기존 `video_prompt_dictionary.md`는 즉梦/Seedance 운용 원칙과 단어 사전의 기본 기준으로 유지하고, 이 파일은 **플랫폼 전환, 대사 중복, 실패 단어 보강, 结界 표현**을 추가 관리한다.

---

## 1. 플랫폼 전환 출력 규칙

사용자가 어떤 플랫폼에서 생성하는지 말하면, 그 플랫폼에 맞춰 프롬프트 형식을 바꾼다.

```text
사용자가 “지멍 / 即梦 / Seedance”라고 말하면:
- 짧고 압축된 연속 중국어 프롬프트를 우선한다.
- 한 컷 안의 동작은 2-3개 이하로 줄인다.
- 부정문은 과하지 않게, 핵심 실패 방지만 넣는다.
- Markdown식 긴 구조 설명은 줄이고, 실제 투입용 한 덩어리 문장을 제공한다.
```

```text
사용자가 “Liblib / LibTV”라고 말하면:
- 구조화된 긴 프롬프트를 제공한다.
- 【参考素材】、【角色锁定】、【场景锁定】、【镜头与时间分割】、【动作 / 表演】、【对白】、【画质稳定】、【负面提示】 섹션을 사용한다.
- 即梦用 “最终投喂压缩版”은 제공하지 않는다.
- 시간 분할과 배우 연기 지시를 더 자세히 써도 된다.
```

```text
사용자가 플랫폼을 말하지 않으면:
- 최근에 사용한다고 말한 플랫폼을 우선한다.
- 최근 플랫폼이 불명확하면, 프롬프트 작성 전 어느 플랫폼인지 짧게 확인한다.
```

## 2. Liblib / LibTV 전용 프롬프트 구조

Liblib / LibTV에서는 즉梦 압축문이 필요 없다. 아래 구조를 기본으로 사용한다.

```text
【Liblib / LibTV 视频生成提示词】

《作品名》第X集｜CUT-XX｜标题｜秒数

【参考素材】
角色参考 / 道具参考 / 导演参考

【角色锁定】
人物脸、发型、服装、气质固定。
动物或道具形态固定。

【场景锁定】
地点、背景、风向、光线、空间关系固定。

【镜头与时间分割】
0-X秒：动作一。
X-X秒：动作二。
X-X秒：台词或眼神变化。

【动作 / 表演】
只写可见动作、视线、表情、身体状态。

【对白】
台词只在这里写一次。

【画质稳定】
真人电影感、人物清晰、不变形、服装材质自然。

【负面提示】
只放本CUT最容易错的内容。
```

## 3. 即梦 / Seedance 전용 출력 규칙

即梦/Seedance에서는 너무 긴 감독판 프롬이 오히려 흔들릴 수 있다. 최종 투입용은 가능한 한 **연속 중국어 한 단락**으로 정리한다.

```text
参考素材 + 素材锁定 + 场景 + 2-3个连续动作 + 单一镜头功能 + 台词一次 + 画质稳定 + 少量关键负面约束
```

즉梦용에서는 아래를 피한다.

```text
- 같은 대사를 여러 섹션에 반복하기
- 너무 많은 Negative Prompt 나열
- 영어 키워드 과다
- 3개 이상 큰 동작 겹치기
- 복합 운镜: 推镜+横移+旋转+变焦
- 사용자가 Liblib/LibTV라고 말했는데 “即梦最终投喂压缩版”을 붙이는 것
```

## 4. 대사 중복 방지 공용 규칙

영상 생성 프롬프트에서 대사는 반드시 한 곳에만 쓴다. 같은 문장을 여러 섹션에 반복하면 모델이 대사를 중복 발화하거나 같은 장면 블록을 반복할 수 있다.

```text
对白只能出现一次。
不要在【本CUT重点】、【分镜执行】、【对白】、【最终压缩版】里重复写同一句。
如果需要说明台词情绪，只描述语气，不重复原句。
最终生成用文本中只保留一份对白。
```

대사가 있는 컷에서 사용할 고정문:

```text
本CUT只有这一段对白，只朗读一次。
不要重复朗读。
不要拆成多次对白。
不要在画面中显示字幕。
```

대사 전/후 입모양도 분리한다.

```text
대사 전：입 닫힘, 무대사.
대사 중：입이 열리고 해당 대사 1회.
대사 후：입이 자연스럽게 닫힘, 추가 대사 없음.
```

## 6. 结界 / 阵法 / 无形阻力 표현 사전

昆仑结界, 遮气阵, 阵法, 封印 같은 이상 현상은 과한 특수효과보다 **공기·정적·시선·저항감**으로 표현하는 것이 안정적이다.

### 좋은 표현

```text
空气阻力
看不见的阻力
空气被压住
空气轻微扭曲
冷灰半透明空气幕
低饱和冷灰光痕
轻微透明空气波纹
像透明玻璃上的水纹
空气被压弯
边界只通过身体被推回表现
异常通过声音、停顿、视线和环境安静表现
```

### 피해야 하는 표현

```text
能量门户
光圈
传送门
科幻门
蓝白电光
闪电
落雷
电流
魔法阵
发光符咒
强光屏障
旋涡
时空裂缝
跨越时空
```

### 교정문

```text
昆仑边界不是门户，不是传送门，不是光圈。
它只是冷灰半透明空气幕，像空气被压出一层淡淡波纹。
结界不发射攻击，不出现雷电，不出现电流。
陆沉舟是被看不见的空气阻力推回，不是自己摔倒。
```

## 7. 女王印 / 道具 표현 규칙

女王印은 자주 宝盒, antique box, gift box로 바뀌므로 보이는 컷과 안 보이는 컷을 명확히 나눈다.

```text
女王印 보이는 컷:
必须参考@女王印参考图，保持古老印章信物形态。
它是印章，不是盒子，不是宝盒，不是宝箱，不是首饰。
```

```text
女王印 안 보이는 컷:
本CUT不显示女王印，不显示盒子，不显示任何手持道具。
```

특히 얼굴 클로즈업, 손 클로즈업, 대사 컷에서는 도구를 일부러 빼는 것이 안정적일 수 있다.

## 8. 실제 이동 / 原地跑 방지 공용문

액션 이동 컷에서 캐릭터가 제자리 뛰기처럼 나오면 아래 문장을 사용한다.

```text
角色必须真实向前移动，不是在原地摆动手脚。
身体必须离开起始位置，脚步真实踩过地面。
地面碎石和远山必须产生视差变化，证明角色完成位移。
起点和终点必须明显不同。
```

걷거나 떠나는 컷에서도 사용한다.

```text
角色必须从原位置真实走到目标位置，距离逐渐拉开。
不能突然出现在目标位置，不能瞬移。
镜头要表现从起点到终点的连续移动。
```

## 9. 플랫폼별 출력 체크리스트

프롬프트를 주기 전 아래를 확인한다.

```text
1. 사용자가 이번 생성 플랫폼을 말했는가? 即梦인지 Liblib/LibTV인지 확인.
2. 플랫폼에 맞는 형식인가? 即梦이면 압축, Liblib이면 구조형.
3. 같은 대사가 두 번 이상 쓰였는가?
4. 감독参考图의 글자/화살표/흑백画风을 무시하라고 썼는가?
5. 캐릭터 참고는 실제 등장 인물만 넣었는가?
6. 女王印이 보이는 컷인지 숨겨야 하는 컷인지 정했는가?
7. 결계가 portal/lightning/energy door로 바뀔 단어가 들어갔는가?
8. 영어 미화 단어가 캐릭터를 일반 AI 인물로 바꾸지 않는가?
9. 액션/이동 컷에서 실제 시작점과 도착점이 있는가?
10. 손이 보이는 컷에서 손 개수와 도구 유무를 잠갔는가?
```

## 10. 적용 우선순위

새 프롬프트 작성 시 우선순위는 다음과 같다.

```text
1. 사용자 최신 지시: 이번 플랫폼, 이번 컷, 이번 수정사항.
2. 현재 컷 분镜 / 분할 참고도.
3. 생성 실패 분석 자료.
4. 캐릭터/도구 참고 이미지.
5. video_prompt_dictionary.md 기본 원칙.
6. episode19_acting_split_timing_rules.md 시간 분할 경험값.
7. 이 파일의 플랫폼 전환 / 실패 단어 / 结界 표현 규칙.
```
