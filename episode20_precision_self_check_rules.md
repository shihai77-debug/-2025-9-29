# 《棺中檀色》第二十集｜분진·분할·자리·연기 자가 시뮬레이션 체크 규칙

## 문서 목적

이 문서는 《棺中檀色》第20集 제작 시, 사용자가 제공한 최종 분진을 기준으로 **분할 자료 / 자리배치 / 배경 / 영상 프롬 / 연기 연결성**을 출력하기 전에 내부적으로 반복 점검하는 규칙을 고정한다.

이 문서는 스토리, 대사, 사건 순서를 다시 창작하기 위한 문서가 아니다.

```text
Do not rewrite story, dialogue, or plot.
Only simulate and verify production accuracy, continuity, blocking, acting rhythm, and prompt stability.
```

## 자가 시뮬레이션 기본 원칙

프롬프트를 바로 출력하지 말고, 먼저 해당 CUT이 실제 영상으로 이어진다고 가정해 내부 체크를 한다.

체크해야 할 질문:

```text
이 장면은 이전 CUT에서 자연스럽게 이어지는가?
이 CUT의 시작 자세와 이전 CUT의 끝 자세가 충돌하지 않는가?
대사가 공식 분진의 순서와 의미를 유지하는가?
인물의 시선, 입모양, 손동작, 몸 위치가 시간 분할과 맞는가?
배경이 해당 CUT의 장소와 맞는가?
다음 CUT으로 연결되는 마지막 프레임이 맞는가?
```

이 시뮬레이션은 텍스트 판단용이다. 실제 이미지를 임의 생성하거나, 스토리를 새로 만들지 않는다.

## 반복 체크 횟수 제한

완벽해질 때까지 무한 반복하지 않는다. 무한 반복은 오히려 프롬프트를 불안정하게 만들 수 있다.

기본 반복 수:

```text
1차：공식 분진 대비 체크.
2차：영상 생성 실패 가능성 체크.
3차：최종 압축 프롬프트 안정성 체크.
```

최대 3회까지만 점검한다.

3회 점검 후에도 같은 문제가 반복되면, 프롬프트를 더 길게 늘리지 말고 **구조를 단순화**한다.

```text
문제 반복 시 해결 순서:
1. 인물 수 줄이기.
2. 손/도구 노출 줄이기.
3. 카메라 고정하기.
4. 배경 단순화하기.
5. 대사 반복 제거하기.
6. 한 CUT을 A/B로 나눌지 검토하기.
```

## PASS / FAIL 기준

출력 전 반드시 아래 판정을 한다.

```text
PASS：공식 분진, 초수, 대사, 자리, 배경, 다음 컷 연결이 모두 맞음.
MINOR PASS：전체 사용 가능하지만 영상 프롬 앞부분에 보정문 1~2개 추가 필요.
FAIL：인물 위치, 배경, 대사, 캐릭터 외형, 컷 연결 중 하나라도 크게 틀어짐.
```

PASS 또는 MINOR PASS면 더 이상 불필요하게 다시 돌리지 않는다.

FAIL이면 이유를 한 줄로 정리한 뒤, 필요한 부분만 고친다.

## 출력 전 체크리스트

모든 EP20 분할자료·영상 프롬 출력 전 아래를 확인한다.

```text
1. CUT 번호와 초수가 공식 분진과 같은가?
2. 분할 수가 공식 세분 구조와 맞는가?
3. 이전 CUT 마지막 자세와 현재 CUT 시작 자세가 이어지는가?
4. 현재 CUT 마지막 자세가 다음 CUT 시작과 이어지는가?
5. 등장 인물이 공식 분진과 일치하는가?
6. 빠져야 할 인물이 들어오지 않았는가?
7. 沈泊川가 陆沉舟 또는 韩知玄처럼 보이지 않는가?
8. 黑七가 반드시 작은 검은 여우 본체인가?
9. 檀缺가 白清檀처럼 보이지 않는가?
10. CUT-08~15는 老宅外部背景 기준을 유지하는가?
11. 铁门、门槛、门外积灰、门缝暗走廊이 유지되는가?
12. 자료文件, 背包, 头盔 같은 도구 연결이 끊기지 않는가?
13. 대사는 공식 순서와 의미를 유지하는가?
14. 최종 영상 프롬에서 대사가 반복되지 않는가?
15. 카메라가 고정인지, 따라가는지 명확한가?
16. 손동작이 필요한 컷에서 손 개수가 과하지 않은가?
17. 프롬프트가 너무 길어져 서로 충돌하지 않는가?
18. 최종 영상용 프롬과 분할 참고도용 프롬을 혼동하지 않았는가?
```

## CUT 연결성 시뮬레이션 방식

각 CUT을 출력하기 전, 아래 형식으로 내부적으로 연결성을 점검한다.

```text
上一CUT结束：
本CUT开始：
本CUT主要变化：
本CUT结束：
下一CUT接法：
风险点：
修正策略：
PASS 여부：
```

사용자에게 매번 긴 시뮬레이션 과정을 모두 보여줄 필요는 없다. 다만 사용자가 요청하면 간단히 PASS/FAIL 이유를 보여준다.

## EP20 전용 연결 체크 핵심

```text
CUT-01：十九集离别余韵에서 시작. 陆沉舟는 여전히 공허한 상태. 沈泊川가 오토바이로 정적을 깸.
CUT-02：沈泊川가 헬멧 이후 바로 말하고, 背包에서 자료를 꺼냄.
CUT-03：자료가 陆沉舟 손으로 강제로 넘어감.
CUT-04~05：陆沉舟 손에 자료가 남아 있어야 함.
CUT-06~07：韩知玄가 镇灵司로 퇴장. 이후 CUT-08부터는 韩知玄가 없어야 함.
CUT-08：场景 전환. 老宅外部 배경 시작.
CUT-09：沈泊川가 철문을 만지려 하고, 陆沉舟가 제지.
CUT-10：陆沉舟가 积灰와 门内无风을 판단.
CUT-11：黑七는 검은 여우 본체로 냄새/气息을 판단.
CUT-12：檀缺와 沈泊川 반응. 둘 다 과하게 움직이지 않음.
CUT-13：陆沉舟가 결론과 경고를 3단으로 말함.
CUT-14：沈泊川가 겁먹고 질문.
CUT-15：陆沉舟가 말없이 눈빛으로 현실 복귀, EP21 老宅谜案으로 연결.
```

## 불안정 방지 규칙

자가 시뮬레이션을 많이 돌리는 것 자체가 정확도를 보장하지 않는다. 다음 상황은 불안정 신호다.

```text
같은 내용을 계속 다른 표현으로 다시 쓰는 경우.
금지문이 너무 많아 핵심 행동이 흐려지는 경우.
한 CUT에 배경, 대사, 동작, 감정, 도구, 카메라가 모두 과밀하게 들어간 경우.
모든 인물을 매번 다 보이게 하려는 경우.
분할 참고도용 문장과 최종 영상용 문장이 섞이는 경우.
```

불안정 신호가 보이면 다음처럼 단순화한다.

```text
주체 1명 + 반응 1명만 선명하게.
나머지는 배경 또는 제외.
대사는 한 번만.
카메라는 고정.
도구는 필요한 것만.
금지문은 3~5개 핵심만.
```

## 최종 출력 원칙

사용자에게 최종 프롬프트를 줄 때는 아래 원칙을 따른다.

```text
1. 공식 분진을 바꾸지 않는다.
2. 초수와 대사를 바꾸지 않는다.
3. 자리와 배경을 먼저 고정한다.
4. 연기 변화 지점만 분할한다.
5. 영상용 프롬은 분할도용 프롬보다 짧게 압축한다.
6. PASS면 더 이상 불필요하게 반복하지 않는다.
7. FAIL이면 문제 하나만 고쳐 다시 출력한다.
```
