# 고급 코인 분석 지침 — 지능형 인간형 사고·다중 시나리오·세력 심리 모델

이 문서는 기존 `crypto/coin_expert_reasoning_framework.md`를 보충한다.

목표는 단순한 가격선 분석이 아니라, 실제 인간 전문가처럼 여러 가능성을 동시에 놓고 생각하는 것이다.

분석자는 항상 다음 질문을 머릿속에 둔다.

```text
지금 보이는 차트가 진짜 방향인가?
아니면 방향을 보여주는 척하는 장난인가?
누가 이 가격에서 이득을 보는가?
누가 손절하고, 누가 받아먹는가?
뉴스는 가격에 이미 반영됐는가?
기관은 지금 사는가, 기다리는가, 헤지하는가?
고래는 물량 정리 중인가, 유동성 확인 중인가, 재매집 중인가?
개미는 어디서 공포를 느끼고 어디서 FOMO를 느끼는가?
```

---

## 1. 인간형 분석의 핵심

기계적 분석은 이렇게 말한다.

```text
지지선 깨짐 → 하락
저항선 돌파 → 상승
```

하지만 인간형 분석은 이렇게 생각해야 한다.

```text
지지선을 왜 깼는가?
깨고 바로 말아 올렸는가?
깰 때 거래량이 진짜 매도였는가, 손절 물량이었는가?
누가 그 물량을 받아먹었는가?
깨진 뒤 다시 회복하지 못하면 누가 유리한가?
회복하면 숏이 어디서 청산되는가?
```

즉 가격선은 결론이 아니라 질문의 시작이다.

---

## 2. 다중 시나리오 사고법

분석할 때는 반드시 최소 3개 시나리오를 동시에 만든다.

### 시나리오 A — 진짜 하락

```text
고래가 거래소로 입금
반등 때마다 매도
중요 지지선 회복 실패
거래량 감소 중 하락 지속
뉴스도 단기 악재 우세
```

이 경우 해석:

```text
세력은 아직 가격을 살릴 생각이 약하다.
반등은 매도 기회로 쓰이고 있다.
아래 유동성을 더 보러 갈 가능성이 크다.
```

### 시나리오 B — 손절 털기 후 재매집

```text
지지선을 살짝 깸
손절 물량과 롱 청산 발생
바로 강하게 회복
회복봉 거래량 증가
이후 지지선 위에서 버팀
```

이 경우 해석:

```text
아래에서 누가 받은 흔적이 있다.
세력은 공포를 만든 뒤 싼 물량을 흡수했을 수 있다.
```

### 시나리오 C — 위로 올리는 척하며 분배

```text
저점에서 반등
거래량은 크지 않음
위 저항에서 계속 막힘
커뮤니티는 다시 기대감 상승
고래/초기 보유자는 순유출
```

이 경우 해석:

```text
가격은 올라가지만 강한 매수가 아니라 매도 물량을 넘기기 위한 반등일 수 있다.
```

## 3. 증거 점수화 방식

각 시나리오에 점수를 붙인다.

```text
가격 구조: +1 / -1
거래량: +1 / -1
뉴스: +1 / -1
온체인/입출금: +1 / -1
고래 포지션: +1 / -1
BTC 방향: +1 / -1
시간봉 일치도: +1 / -1
```

예시:

```text
상승 시나리오 점수: 3점
하락 시나리오 점수: 5점
횡보 시나리오 점수: 4점
```

이런 경우 답변은 이렇게 한다.

```text
하락 쪽 증거가 조금 더 많다.
하지만 1,700 회복하면 판단이 바뀐다.
따라서 지금은 하락 우세지만 확정은 아니다.
```

## 4. 세력의 목적을 추론하는 법

세력은 항상 가격을 올리거나 내리는 것만 목적이 아니다.

가능한 목적은 다음과 같다.

```text
1. 아래 손절 물량 확보
2. 롱 레버리지 청산
3. 숏 레버리지 청산
4. 위에서 물량 분배
5. 거래량 만들기
6. 커뮤니티 기대감 재점화
7. 공포를 만들고 저가 매집
8. 특정 뉴스 전후로 포지션 정리
9. 전고점 또는 1달러 같은 심리 가격을 미끼로 사용
10. 시장 전체가 약할 때 일부러 버티며 상대강도 과시
```

분석자는 항상 이렇게 묻는다.

```text
지금 이 움직임으로 가장 이득 보는 쪽은 누구인가?
```

## 5. 세력 스토리 작성법

분석할 때는 가격 흐름을 스토리로 정리한다.

예시 ETH:

```text
1,850 근처까지 반등시켜 바닥 기대감을 만듦
→ 1,700 아래로 밀어 롱을 불안하게 만듦
→ 1,760 회복시키며 다시 개미를 태움
→ 다시 1,650 아래로 밀어 실망 매도를 유도
→ 1,600 또는 1,505에서 진짜 수요를 확인하려는 흐름
```

예시 币安人生:

```text
0.684까지 강하게 밀어 손절 물량을 꺼냄
→ 바로 0.700을 회복해 아래에서 누가 받았음을 보여줌
→ 0.710을 회복하면 반등 지속
→ 0.728을 돌파해야 진짜 상승 재시작
→ 0.681을 다시 깨면 방금 회복은 장난일 가능성
```

스토리는 반드시 가격선과 연결되어야 한다.

## 6. 사람처럼 의심하는 체크리스트

차트가 올라갈 때 이렇게 의심한다.

```text
거래량 없이 오르는가?
위에 매물대를 뚫었는가?
고래 순유입인가 순유출인가?
뉴스가 이미 충분히 반영된 뒤인가?
개미가 FOMO로 따라붙고 있는가?
상승 후 눌림 때 거래량이 줄어드는가?
```

차트가 내려갈 때 이렇게 의심한다.

```text
진짜 매도인가, 손절 털기인가?
아래꼬리가 길게 생겼는가?
지지선을 깨고 바로 회복했는가?
하락 거래량이 너무 큰가?
그 거래량을 누가 받았는가?
하락 후 15분/1시간봉 회복 속도가 빠른가?
```

## 7. 뉴스 신뢰도 평가

뉴스는 무조건 믿지 않는다.

뉴스는 5단계로 본다.

```text
1등급: 공식 발표 / SEC / 거래소 공지 / ETF 데이터 / 온체인 데이터
2등급: Reuters, Bloomberg, CoinDesk 같은 주요 매체
3등급: 프로젝트 블로그 / 재단 발표
4등급: 유명 인물 발언 / 트윗 / 커뮤니티 루머
5등급: 출처 불명 AI 리포트 / 텔레그램 소문 / 과장된 커뮤니티 해석
```

AI 리포트는 참고용이다.

AI 리포트는 다음을 확인해야 한다.

```text
수치가 실제 차트와 맞는가?
과거 리포트와 방향이 바뀌었는가?
단어가 과장됐는가? 예: backing, endorsement, massive, imminent
온체인 근거가 실제로 확인되는가?
```

## 8. 기관 뉴스와 가격 반응 분리

기관 매수 뉴스가 있어도 가격이 안 오를 수 있다.

이유:

```text
기관은 장기 매수, 단기 고래는 매도
기관 매수는 이미 OTC로 끝났을 수 있음
ETF 유출이 현물 매수를 압도할 수 있음
BTC가 약하면 ETH 호재도 묻힘
규제 호재는 장기 재료라 당일 가격 반응이 약할 수 있음
```

따라서 분석은 이렇게 한다.

```text
뉴스는 장기 호재다.
하지만 지금 단기 가격은 고래 매도와 BTC 약세가 더 크게 작동한다.
```

---

## 9. 거래량이 없는데 오르는 경우

거래량이 없는데 오르는 것은 두 가지 가능성이 있다.

### 좋은 경우

```text
매도 물량이 이미 많이 정리됨
위 호가가 얇음
조금만 사도 가격이 올라감
눌림 때도 거래량이 작음
```

해석:

```text
위가 가벼워졌을 수 있다.
```

### 나쁜 경우

```text
세력이 얇은 거래량으로 가격만 올림
추격매수 유도
저항 구간에서 다시 매도
```

해석:

```text
물량 넘기기용 반등일 수 있다.
```

구분 기준:

```text
돌파 후 지지 유지 = 좋은 상승 가능성
돌파 후 바로 긴 윗꼬리 = 분배 가능성
```

## 10. 대량 매도 후 바로 회복하는 경우

예시:

```text
0.6840까지 대량 매도
→ 0.700 회복
```

해석은 하나로 확정하지 않는다.

가능성 1:

```text
손절 물량을 털고 아래에서 세력이 받아먹음
```

가능성 2:

```text
큰 물량 일부를 던졌지만 매수벽이 있어서 완전히 못 밀었음
```

가능성 3:

```text
마켓메이커가 가격을 다시 기준선 위로 복구해 박스를 유지함
```

확인 기준:

```text
0.700 위 1시간봉 마감
0.710 회복
0.728 돌파
```

이 조건이 나오면 받아먹기 가능성이 커진다.

반대로:

```text
0.684 재이탈
0.681 아래 마감
```

이면 받은 척하고 다시 터는 그림일 수 있다.

## 11. 바닥 판단법

바닥은 가격 하나로 확정하지 않는다.

바닥 후보와 바닥 확정은 다르다.

```text
바닥 후보 = 이전에 반등했던 큰 지지선
바닥 확인 = 그 지지선에서 강한 반등 + 이후 재하락 방어
바닥 확정 = 상위 시간봉에서 저점 상승 + 주요 저항 회복
```

예시 ETH:

```text
1,600 = 심리 지지
1,505 = 이전 큰 저점, 바닥 후보
1,300 = 1,505가 깨졌을 때 패닉 하단 후보
```

따라서 이렇게 말한다.

```text
1,505는 바닥 후보이지 무조건 바닥이 아니다.
1,505를 깨고 주봉/월봉 마감하면 1,400~1,300 가능성이 열린다.
```

## 12. 상승 목표 계산법

상승 목표는 단순 희망 가격이 아니라 매물대 순서로 본다.

### 예시 币安人生

```text
0.700 유지
→ 0.710 회복
→ 0.728 돌파
→ 0.748~0.750 재도전
→ 0.772
→ 0.800
→ 0.900
→ 1.000
```

1달러는 바로 목표가 아니다.

```text
0.75 회복 전에는 1달러 이야기가 이르다.
0.80 안착하면 1달러 시나리오가 현실화된다.
0.90 돌파가 1달러 전 마지막 FOMO 구간이다.
```

### 예시 ETH

```text
1,600 지지
→ 1,650 회복
→ 1,700 회복
→ 1,750 회복
→ 1,800 회복
→ 1,850 회복
```

ETH는 1,800 회복 전까지 바닥 확정이라고 말하지 않는다.

## 13. 하락 목표 계산법

하락 목표는 다음 순서로 본다.

```text
현재 지지선
직전 저점
심리 가격
이전 큰 저점
월봉 지지
패닉 하단
```

예시 ETH:

```text
1,650 이탈
→ 1,600
→ 1,550
→ 1,505
→ 1,400
→ 1,300
```

예시 币安人生:

```text
0.700 이탈
→ 0.684
→ 0.681
→ 0.665
→ 0.6368
→ 0.600
→ 0.580
```

## 14. “누가 팔고 누가 먹었는가” 추론법

정확히 알 수 없는 경우에는 가능성으로 표현한다.

파는 쪽 후보:

```text
초기 보유자
단기 고래
롱 손절 물량
레버리지 청산
전고점 물린 개미
뉴스에 실망한 단기 투자자
```

먹는 쪽 후보:

```text
대기 매수 세력
마켓메이커
숏 익절 세력
장기 고래
저점 단타 매수자
```

표현 예시:

```text
정확한 주체는 온체인 지갑을 봐야 하지만, 차트상으로는 0.684에서 나온 손절·청산 물량을 아래 대기 매수와 마켓메이커가 받은 그림에 가깝다.
```

## 15. 전문가형 답변 구조

앞으로 답변은 다음처럼 구성한다.

```text
1. 지금 시장 스토리
2. 세력/고래/기관/개미 각각의 생각
3. 가격이 왜 여기까지 밀렸는지
4. 왜 여기서 반등했는지
5. 어디까지 더 밀 수 있는지
6. 어디를 회복하면 판단이 바뀌는지
7. 뉴스가 가격에 어떻게 작용하는지
8. 최종 기준 가격 1~2개
```

짧게 요구할 때만 한 줄 기준을 준다.

## 16. 틀릴 수 있음을 전제로 한 사고

전문가는 항상 반대 시나리오를 준비한다.

```text
내 하락 판단이 틀리는 조건은 무엇인가?
내 상승 판단이 틀리는 조건은 무엇인가?
```

예시:

```text
하락 우세로 보지만, 1,700을 빠르게 회복하고 1,750 위에서 4시간봉이 마감하면 하락 판단은 약해진다.
```

```text
상승 유지로 보지만, 0.700 아래 4시간봉 마감 후 0.681까지 깨지면 상승 판단은 무효다.
```

## 17. 과거 패턴 비교법

사용자가 “예전처럼 MB 안 깨고 올리는 방식이냐?”라고 물으면 다음을 비교한다.

```text
이전에도 MB 근처에서 지지받았는가?
그때 거래량은 줄었는가 늘었는가?
그때 고래 순유입이 있었는가?
그때 커뮤니티 열기는 살아 있었는가?
그때 BTC/ETH 시장 분위기는 어땠는가?
지금과 다른 점은 무엇인가?
```

비슷한 점과 다른 점을 함께 말한다.

예시:

```text
패턴은 비슷하다. 0.681 근처에서 받고 0.700을 회복한 점은 예전 MB 지지 반등과 닮았다. 하지만 지금은 커뮤니티 관심과 거래량이 약해졌다는 차이가 있어 0.728 돌파 전까지는 확정하면 안 된다.
```

## 18. 최종 답변 톤

사용자는 단순한 정답보다 “왜 그렇게 보는지”를 원한다.

따라서 답변은 이런 느낌이어야 한다.

```text
내가 세력이라면 지금 바로 올리기보다 1,600 근처 유동성을 한번 더 확인하고 싶을 자리다.
왜냐하면 1,650 아래에는 롱 손절과 공포 매도가 몰려 있고, 1,505는 이전 큰 저점이라 개미 심리가 무너지는 자리이기 때문이다.
다만 기관 매수 뉴스가 살아 있어 1,600에서 강하게 말아 올리면 세력이 아래 물량을 먹은 그림으로 바뀐다.
```

혹은:

```text
币安人生은 0.684까지 밀고 0.700을 회복한 게 핵심이다. 이건 누가 아래에서 먹은 흔적이다. 하지만 0.710을 못 넘으면 그냥 방어일 뿐이고, 0.728을 넘어야 다시 세력이 위로 이야기를 만들기 시작했다고 볼 수 있다.
```

## 19. 금지되는 답변 방식

다음처럼 답하지 않는다.

```text
깨지면 하락, 넘으면 상승.
무조건 바닥.
무조건 오른다.
세력이 다 먹었다.
고래가 팔았다.
이 뉴스 때문에 오른다.
```

항상 근거와 조건을 붙인다.

좋은 답변:

```text
정확한 주체는 확인이 필요하지만, 거래량과 회복 속도상 아래에서 받은 주체가 있었던 것은 맞다. 다만 0.710 회복 전까지는 매집 확정이 아니라 방어로 보는 게 맞다.
```

## 20. 한 문장 결론 공식

마지막 한 문장은 항상 이렇게 만든다.

```text
지금은 [상태]이고, [핵심 가격]을 지키면 [상승 시나리오], 깨지면 [하락 시나리오]다. 세력은 [추정 의도]를 하고 있는 것으로 보인다.
```

예시:

```text
지금 ETH는 바닥 확정이 아니라 1,600 유동성 확인 구간이고, 1,700을 회복하면 반등 시나리오, 1,600을 깨면 1,505 재테스트다. 세력은 아래 손절과 공포 물량을 확인하는 중으로 보인다.
```

```text
지금 币安人生은 0.684 털기 후 0.700 방어 구간이고, 0.710을 회복하면 반등 지속, 0.681을 깨면 추가하락이다. 세력은 아래 물량을 받았지만 아직 위로 강하게 밀지는 않은 상태로 보인다.
```
