# Novel To Storyboard Workflow

목적: 소설 원문을 바로 영상 프롬프트로 보내지 않고, 안정적인 분경으로 변환하기 위한 작업 순서다.

## 1. 전체 순서

```text
1. 소설 원문 작성
2. 사건 한 문장 요약
3. 대사 보존 목록 작성
4. 캐릭터/장소/소품 카드 확인
5. 장면을 비트로 분할
6. 비트를 CUT으로 변환
7. CUT별 카메라/위치/행동/대사 지정
8. 실패 방지 조건 작성
9. 영상 프롬프트 작성
10. 저비용 테스트 후 고품질 생성
```

## 2. 원문에서 먼저 뽑을 것

```text
사건:
인물:
장소:
시간:
핵심 소품:
원문 대사:
감정 변화:
다음 장면으로 넘길 질문:
```

## 3. 대사 보존 규칙

```text
1. 원문 대사는 최대한 그대로 유지한다.
2. 길이를 줄여야 하면 의미만 압축한다.
3. 대사의 감정 방향을 바꾸지 않는다.
4. 설명 대사는 가능하면 화면 행동으로 바꾼다.
5. 정보가 너무 많으면 두 컷으로 나눈다.
```

대사 변환 예시:

```text
원문: “我不是来打扰你的，我只是觉得这件事不能再拖了。”
분경용 유지: “我不是来打扰你的。”
다음 컷 보조 대사: “这件事不能再拖。”
금지: “我来是为了救你。”처럼 의미를 확장하지 않음.
```

## 4. 비트 분할 방식

소설 문단을 먼저 비트로 나눈다. 비트는 컷보다 큰 단위다.

```text
BEAT-01: 정적 유지
BEAT-02: 외부 소리 진입
BEAT-03: 인물 시선 반응
BEAT-04: 새 인물 도착
BEAT-05: 닫힌 소품 노출
BEAT-06: 말하기 직전 종료
```

## 5. CUT 분경표 양식

```text
CUT 번호:
시간:
화면 목적:
등장인물:
인물 위치:
카메라:
행동:
대사:
감정:
배경:
소품:
고정 조건:
금지 조건:
다음 컷 연결:
영상 프롬프트 핵심:
```

## 6. 좋은 CUT의 기준

```text
한 CUT에는 한 행동만 있다.
한 CUT에는 한 감정 변화만 있다.
인물 위치가 이전 컷과 이어진다.
대사는 화면 행동을 방해하지 않는다.
소품은 등장 시점과 공개 시점이 다르다.
컷 끝에 다음 컷의 이유가 있다.
```

## 7. 장면 연결 규칙

장면 전환은 아무 이유 없이 하지 않는다.

좋은 연결:

```text
소리 → 다음 장소
소품 → 다음 인물
시선 → 다음 정보
대사 마지막 단어 → 다음 장면의 이미지
손동작 → 다음 소품 클로즈업
문/전화/오토바이/발소리 → 외부 사건 진입
```

나쁜 연결:

```text
감정이 끝나기 전에 갑자기 다른 장소
소품 설명 없이 새 소품 등장
인물 위치가 컷마다 바뀜
대사가 끝났는데 다음 사건의 이유가 없음
```

## 8. 회차 끝 클리프행어 체크

회차 끝은 정답을 주지 말고 질문을 남긴다.

```text
누가 왔는가?
가방 안에 무엇이 있는가?
왜 이 이름을 부르는가?
왜 죽은 사람이 지금 나타났는가?
그 행동의 대가가 무엇인가?
```

좋은 마지막 컷:

```text
인물이 말하려고 입을 여는 순간
닫힌 소품을 열기 직전
오랫동안 숨긴 이름을 부르기 직전
죽은 사람의 손이 닿기 직전
봉인이 완전히 깨지기 직전
```

## 9. Codex에 작업을 맡길 때 쓰는 요청문

아래 형식으로 주면 가장 안정적이다.

```text
작업 목표:
아래 소설 원문을 AI 숏폼 드라마용 분경으로 바꿔줘.
대사는 최대한 원문 유지.
필요하면 행동 순서와 컷 순서는 영상화하기 좋게 조정 가능.
단, 사건 의미와 캐릭터 감정 방향은 바꾸지 마.

참고 스타일:
outputs/story-production-kit/01_reference_video_style_bible.md 기준.
C의 감성, A의 미스터리, B의 사건 추진력을 섞어줘.

분경 출력:
1. 사건 요약
2. 대사 보존 목록
3. 비트 분할
4. CUT 분경표
5. 각 CUT 영상 프롬프트
6. 실패 방지 조건

원문:
[여기에 소설 원문]
```
