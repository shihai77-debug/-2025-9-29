# Story Density No Drag Execution Gate

목적: 소설, 분경, 영상 프롬프트가 질질 끌리는 문제를 막기 위한 강제 실행 게이트다. 이 문서는 조언이 아니라 작업 전후 필수 검사다.

핵심:

```text
느린 장면은 가능하다.
멈춘 장면은 금지다.

감정 여백은 가능하다.
정보 없는 반복은 삭제한다.
```

## 1. 작업 전 필수 질문

소설/분경을 쓰기 전에 반드시 아래를 먼저 답한다.

```text
이번 장면의 첫 질문은 무엇인가?
이번 장면에서 바뀌는 상태는 무엇인가?
이번 장면 끝에 남는 새 질문은 무엇인가?
이 장면을 삭제하면 다음 장면 이해가 깨지는가?
```

네 질문 중 하나라도 답이 약하면 장면을 시작하지 않는다. 먼저 구조를 고친다.

## 2. 한 장면의 필수 변화

한 장면은 아래 중 최소 하나를 반드시 바꿔야 한다.

```text
정보 변화
위험 변화
감정 변화
관계 변화
공간 상태 변화
소품 상태 변화
선택 압박
다음 화 질문
```

아무것도 바뀌지 않으면 삭제하거나 다른 장면과 합친다.

## 3. 8초 / 20초 / 40초 게이트

숏폼 기준으로 아래 리듬을 지킨다.

```text
0~3초: 질문 생성
8초 이내: 첫 반응 또는 첫 이상 현상
20초 이내: 새 정보 또는 오해 깨기
40초 이내: 위험/선택/반전 중 하나 발생
마지막 5초: 다음 컷 또는 다음 화 질문
```

40초 동안 인물들이 같은 장소에서 같은 걱정만 하면 실패다.

## 4. 대사 반복 삭제 규칙

아래 대사는 삭제 또는 수정한다.

```text
이미 관객이 아는 정보를 다시 말하는 대사
같은 질문을 다른 말로 반복하는 대사
인물이 "어떻게 하지?"만 반복하는 대사
주인공이 같은 판단을 두 번 설명하는 대사
黑七/檀缺/沈泊川가 리액션만 하고 장면 기능이 없는 대사
```

유지 가능한 대사:

```text
새 단서
새 위험
새 감정 균열
캐릭터만 할 수 있는 말
다음 행동을 바꾸는 말
```

## 5. KEEP / DELETE / COMPRESS / MOVE 표

소설 초안 후 반드시 아래 표를 만든다.

```markdown
| Beat | Function | Decision | Reason |
|---|---|---|---|
| 문 안쪽 숨소리 | 첫 질문 | KEEP | 20화 끝에서 바로 이어지는 훅 |
| 같은 문 앞 불안 반복 | 정보 없음 | DELETE | 이미 전달된 감정 반복 |
| 陆沉舟의 局 설명 | 세계관 정보 | COMPRESS | 화면 단서 + 한두 문장으로 분할 |
| 감정 여백 | 정서 잔상 | MOVE | 클라이맥스 뒤로 이동 |
```

판정 기준:

```text
KEEP = 새 정보/위험/감정/선택이 있다.
DELETE = 없어도 이야기 이해가 같다.
COMPRESS = 필요하지만 길다.
MOVE = 위치가 나빠서 리듬을 죽인다.
```

이 표 없이 최종 소설/분경으로 넘어가지 않는다.

## 6. 질질 끄는 장면의 대표 증상

아래가 보이면 즉시 수정한다.

```text
문 앞에서 오래 말하지만 문/집/인물 상태가 안 변함
인물이 계속 놀라기만 함
설정 설명 후 행동이 없음
이미 나온 감정을 다른 컷에서 다시 반복
장면 끝 질문이 장면 시작 질문과 같음
대사는 많은데 선택이 없음
소품은 보이는데 기능이 없음
```

## 7. 21화 전용 압축 기준

21화는 老宅 문 앞에서 시작하므로 특히 조심한다.

필수:

```text
20화 마지막 문 앞 긴장에서 바로 이어진다.
첫 3초에 문 안쪽 반응이 있어야 한다.
10초 안에 黑七 또는 沈泊川의 반응이 상태를 바꿔야 한다.
20초 안에 "귀신이 아니다"보다 더 구체적인 공간 규칙이 나와야 한다.
40초 안에 문을 열지 말아야 하는 이유가 행동/소품으로 증명돼야 한다.
마지막에는 老宅 안쪽의 새 질문이 남아야 한다.
```

금지:

```text
20화 정보를 다시 설명
문 앞에서 같은 불안을 반복
沈泊川를 "怎么办" 리액션 담당으로만 사용
黑七를 냄새 설명 담당으로만 사용
檀缺를 독설 담당으로만 사용
陆沉舟가 긴 이론 강의를 함
```

## 8. 최종 삭제 질문

최종본 직전 반드시 묻는다.

```text
이 문단이 없어도 다음 문단이 이해되는가?
이 대사가 없어도 감정과 정보가 유지되는가?
이 컷이 없어도 사건이 전진하는가?
이 장면이 시작 질문과 다른 질문으로 끝나는가?
```

답이 "그대로 이해된다"면 삭제한다.

## 9. 한 줄 명령

Codex/GitHub 작업자가 질질 끌 때 아래 문장을 적용한다.

```text
최종 출력 전에 Story Density Gate를 실행해. 모든 beat를 KEEP / DELETE / COMPRESS / MOVE로 판정하고, DELETE와 COMPRESS를 실제 반영한 뒤 다시 써.
```
