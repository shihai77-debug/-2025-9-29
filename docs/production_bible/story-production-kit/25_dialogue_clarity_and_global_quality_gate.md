# Dialogue Clarity And Global Quality Gate

목적: 대사를 너무 짧게 잘라 시청자가 내용을 이해하지 못하는 문제를 막고, 소설/분경/영상 프롬프트 전체를 최종 검사해 틀린 부분은 삭제 또는 수정하는 기준을 만든다.

핵심:

```text
짧은 대사 = 좋은 대사가 아니다.
이해되는 대사 = 좋은 대사의 첫 조건이다.
캐릭터다운 대사 = 좋은 대사의 두 번째 조건이다.
스토리를 앞으로 밀어주는 대사 = 좋은 대사의 세 번째 조건이다.
대사보다 먼저 장면 행동을 고정하는 것이 시작 조건이다.
```

## 0. 대사 작성 전 행동 게이트

대사를 쓰기 전에 반드시 장면의 실제 행동 흐름을 먼저 쓴다.

```text
누가 손을 뻗는가?
누가 막는가?
누가 멈추는가?
누가 물러나는가?
무엇이 변하는가?
어떤 소리/냄새/빛/물건이 반응하는가?
그 행동 때문에 다음 컷이 어떻게 바뀌는가?
```

행동 흐름 없이 정보만 전달하는 대사는 금지한다.

작성 순서:

```text
1. 주제와 주체를 한 줄로 잠근다.
2. 장면의 실제 행동 흐름을 먼저 쓴다.
3. 위험 발생 -> 인물 반응 -> 행동 제지 -> 단서 확인 -> 주인공 판단 -> 선택 변화 순서로 장면을 배열한다.
4. 각 인물이 지금 말하지 않으면 장면이 움직이지 않는지 검사한다.
5. 통과한 말만 대사로 쓴다.
```

주제/주체 잠금:

```text
각 화 작성 전, 주제와 주체를 한 줄로 잠근다.
작성 중 모든 장면과 대사는 이 주제/주체를 벗어나면 삭제한다.
조연은 주체를 밀어주는 역할만 한다.
조연이 주체보다 사건을 더 많이 설명하거나 끌고 가면 수정한다.
```

역할 순번 반복 금지:

```text
같은 장면 안에서 인물 발화 순서를 반복하지 않는다.
沈泊川 질문 -> 黑七 감각 -> 檀缺 보정 -> 陆沉舟 정리 구조를 두 번 이상 반복하면 기계 대사로 판정하고 폐기한다.
```

정보 순서 금지:

```text
스토리를 정보 순서로 배열하지 않는다.
소리 -> 냄새 -> 阵痕 -> 法阵 설명 순서로 기계 배치하지 않는다.
위험 발생 -> 인물 반응 -> 행동 제지 -> 단서 확인 -> 주인공 판단 -> 선택 변화 순서로 쓴다.
```

말할 이유 검사:

```text
각 대사는 "지금 이 인물이 말하지 않으면 장면이 움직이지 않는가?"를 통과해야 한다.
통과하지 못하면 삭제하거나 행동/침묵/시선/소품 반응으로 바꾼다.
```

## 1. 금지: 글자 수로 대사 자르기

아래 방식은 금지한다.

```text
무조건 5글자 안에 말하기
무조건 3단어만 쓰기
무조건 6자 이하로 줄이기
무조건 짧으면 영상에 좋다고 판단하기
```

이 방식이 위험한 이유:

```text
시청자가 사건을 못 알아듣는다.
캐릭터 감정이 사라진다.
세계관 단서가 전달되지 않는다.
대사가 전부 비슷해진다.
영상이 멋있어 보여도 스토리가 비어 보인다.
```

## 2. 대사 길이 기준

대사 길이는 글자 수가 아니라 기능으로 정한다.

짧아도 되는 대사:

```text
이미 화면이 정보를 충분히 보여준 경우
감정이 침묵으로 전달되는 경우
위험 순간의 즉각 반응
인물 성격이 짧은 말로 정확히 드러나는 경우
```

길어야 하는 대사:

```text
새 사건을 처음 설명하는 경우
시청자가 공간 규칙을 이해해야 하는 경우
세계관 룰이 처음 드러나는 경우
인물 관계 변화가 걸린 경우
다음 화 떡밥을 심어야 하는 경우
```

원칙:

```text
화면이 말하면 대사는 줄인다.
화면이 못 말하면 대사가 책임진다.
대사가 길면 반응 컷/소품 컷/침묵으로 나눠 영상 리듬을 만든다.
```

## 3. 시청자 이해 검사

한 장면이 끝났을 때 시청자는 최소한 아래를 알아야 한다.

```text
누가 장면의 중심인가?
그 인물은 왜 여기 있는가?
지금 무엇이 이상한가?
무엇을 하면 위험한가?
다음에 무엇을 봐야 하는가?
```

이 다섯 가지 중 하나라도 안 보이면 수정한다.

## 4. 대사 3중 검사

주요 대사는 아래 세 가지를 동시에 담아야 한다.

```text
캐릭터 성격
현재 감정
스토리 정보/단서/긴장
```

실패 예:

```text
여기는 위험해.
문 열지 마.
귀신이 아니야.
어떻게 하지?
```

이런 대사는 기능만 있고 캐릭터와 감정이 약하다.

개선 방향:

```text
LU: 手别碰门。这里不是怕人进，是等人推。
SHEN: 我刚才差点就推了。你能不能别在我差点送命以后才说？
HEIQI: 这味儿太干净了，干净得像有人把死气洗过一遍。
TAN: 不是没人住。是气不准出来。
```

## 5. 전체 품질 검사표

소설/분경/영상 프롬프트를 넘기기 전에 아래를 검사한다.

```text
1. 이전 화 마지막 대사/행동에서 바로 이어지는가?
2. 첫 3초에 질문이 생기는가?
3. 각 인물이 왜 그 장소에 있는지 분명한가?
4. 주인공이 말없는 장식이 아니라 판단/대사/행동으로 장면을 주도하는가?
5. 주제와 주체가 한 줄로 잠겨 있고 모든 장면이 그 안에 있는가?
6. 대사 전에 실제 행동 흐름이 먼저 설계되었는가?
7. 대사 순서가 역할표가 아니라 현장 위험/감정/행동 압력에서 나오는가?
8. 같은 인물 발화 순서가 두 번 이상 반복되지 않는가?
9. 정보 순서가 아니라 위험 반응 순서로 장면이 움직이는가?
10. 각 대사는 지금 말하지 않으면 장면이 움직이지 않는가?
11. 대사가 너무 짧아서 뜻이 비어 있지 않은가?
12. 같은 대사 패턴이 반복되지 않는가?
13. 8~10초마다 새 정보/위험/감정 변화가 있는가?
14. 설명이 길면 반응 컷/소품 컷/행동으로 나눴는가?
15. 금지된 소품/정보가 너무 일찍 나오지 않는가?
16. 마지막 5초에 다음 화 질문이 남는가?
```

## 6. 삭제/수정 기준

검사 후 아래 기준으로 처리한다.

삭제:

```text
이미 화면이 보여준 정보를 다시 말하는 대사
같은 감정을 반복하는 대사
인물 말투가 아니라 작가 설명처럼 보이는 대사
사건을 앞으로 밀지 않는 컷
같은 장소/같은 감정/같은 정보만 반복하는 컷
```

수정:

```text
내용은 필요한데 말투가 캐릭터답지 않은 대사
정보는 필요한데 너무 길어서 영상 리듬을 죽이는 대사
장면은 필요한데 첫 이미지가 약한 컷
감정은 필요한데 행동/시선/소품 반응이 부족한 컷
```

유지:

```text
캐릭터 감정 전환이 걸린 대사
다음 사건 단서가 들어간 대사
인물 관계를 바꾸는 대사
시청자가 사건을 이해하는 데 필요한 대사
다음 화를 보고 싶게 만드는 마지막 질문
```

## 7. 21화 적용 기준

21화는 老宅 문 앞에서 바로 이어진다. 그래서 대사가 너무 짧으면 사건이 안 보인다.

필요한 정보:

```text
문을 열면 왜 위험한가?
집 안에 귀신이 없는 것 같은데 왜 더 위험한가?
局이 어떤 방식으로 공간을 막고 있는가?
沈泊川는 왜 이 사건을 혼자 처리하지 못했는가?
陆沉舟가 왜 지금 정신을 老宅 쪽으로 돌리는가?
```

21화 주제/주체 잠금:

```text
주제: 法阵封口 / 半松不破 / 引出布阵人
주체: 陆沉舟가 老宅의 封口 구조를 읽고, 완전히 깨지 않고 반쯤 느슨하게 만드는 선택을 주도한다.
```

21화 행동 흐름 우선:

```text
沈泊川가 문고리/문환에 손을 뻗는다.
陆沉舟가 먼저 막는다.
문 안쪽 气/소리/먼지/문턱 중 하나가 반응한다.
인물들이 말하기 전에 몸이 먼저 멈춘다.
단서를 확인한 뒤에야 대사가 나온다.
陆沉舟의 판단은 다음 행동 선택으로 이어진다.
```

이 정보는 한 번에 설명하지 않는다.

분할 방식:

```text
문고리 반응
먼지 방향
바람이 없는 복도
黑七의 냄새 감지
檀缺의 한마디 관찰
沈泊川의 현실적 질문
LU의 단계적 판단
```

## 8. 최종 판정

최종본을 만들기 전에 반드시 아래 질문을 한다.

```text
내가 처음 보는 시청자라면 이 장면을 이해하는가?
이 대사는 이 캐릭터만 할 수 있는 말인가?
이 장면을 5초 줄이면 더 좋아지는가?
이 장면을 삭제해도 이야기가 그대로라면 왜 남겼는가?
이 컷이 다음 컷을 보고 싶게 만드는가?
```

하나라도 답이 약하면 삭제 또는 수정한다.
