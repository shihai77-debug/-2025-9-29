# Character Voice And Auto Detail System

목적: 소설을 만들 때 매번 말투, 행동 디테일, 감정 표현, 장소 묘사를 처음부터 설계하지 않도록 자동 생성 기준을 만든다.

중요:

```text
자동 생성은 세계관/캐릭터를 바꾸는 것이 아니다.
자동 생성은 이미 정해진 캐릭터와 장면 목적 안에서 디테일을 채우는 것이다.
```

## 1. 자동 생성 가능한 것

자동 생성 가능:

```text
짧은 행동 디테일
시선/손동작/호흡
장소의 촉감/냄새/소리
대사 앞뒤의 반응
소품을 만지는 방식
장면 전환용 소리
조연의 기능성 대사
```

자동 생성 금지:

```text
캐릭터 성격 변경
캐릭터 말투 방향 변경
세계관 핵심 설정 추가
주요 관계 변경
죽음/부활/정체 공개
계약/규제 위반 설정
```

## 2. 캐릭터 말투 자동화 규칙

말투는 캐릭터 카드가 최우선이다. 캐릭터 카드가 없으면 임시 말투를 확정하지 말고 `NEEDS CHARACTER VOICE CONFIRMATION`으로 표시한다.

### LU / 陆沉舟

```text
말투: 낮고 짧지만 판단이 정확함.
대사 성향: 감정 설명보다 판정, 제지, 결론.
좋은 대사 기능: "멈춰라", "이건 아니다", "지금 열면 안 된다".
금지: 장황한 자기감정 설명, 과한 농담, 즉각적인 통곡.
```

자동 대사 패턴:

```text
"别动。"
"这里不是闹鬼。"
"有人把气堵死了。"
"现在开门，会出事。"
```

### SHEN / 沈泊川

```text
말투: 젊고 민감함. 가볍게 들어오지만 선을 넘지 않음.
대사 성향: 분위기를 읽고, 정보 전달을 책임짐.
좋은 대사 기능: 농담으로 긴장 완화, 곧바로 본론, 자기가 처리 못하는 이유 인정.
금지: 너무 무례한 캐묻기, 사건을 가볍게만 취급, HAN처럼 공무적으로 말하기.
```

자동 대사 패턴:

```text
"你们这脸色，不像刚打完架，像天塌了一半。"
"我不问私事。今天是真有事。"
"这案子不对劲，我压不住。"
"所以我只能来找你。"
```

### HAN / 韩知玄

```text
말투: 공무적, 절제, 책임 중심.
대사 성향: 보고, 판단, 규정, 책임.
좋은 대사 기능: 镇灵司 관점 정리, 퇴장 이유 명확화.
금지: 감정 과잉, 사적 농담 과다, 책임 회피.
```

자동 대사 패턴:

```text
"昆仑结界异动，我必须回镇灵司上报。"
"此事不能拖。"
"路上小心。"
```

### TAN / 檀缺

```text
말투: 조용하고 짧음. 걱정이 먼저 드러남.
대사 성향: 적게 말하지만 시선과 정지로 감정을 표현.
좋은 대사 기능: LU의 상태 확인, 사건 속 이상함 감지.
금지: 설명 담당으로 전락, 갑자기 과한 수다.
```

자동 대사 패턴:

```text
"你还好吗。"
"他没有缓过来。"
"这屋子……不对。"
```

### HEIQI / 黑七

```text
말투: 본능, 냄새, 낮은 반응 중심.
대사 성향: 냄새/气息/위험 감지.
좋은 대사 기능: 귀신 여부 판단, 보이지 않는 기운 감지.
금지: 사람처럼 장황한 분석, 인간화 과다.
```

자동 대사 패턴:

```text
"没有鬼味。"
"也没有活人的热气。"
"这味道，是被人堵出来的。"
```

## 3. 자동 디테일 생성 규칙

디테일은 장면 목적을 보조해야 한다.

### 감정 정적 디테일

```text
바람이 있어야 할 곳에 바람이 없음.
누군가 손에 든 물건을 내려놓지 못함.
눈은 움직이지만 몸은 움직이지 않음.
말하려다 숨만 삼킴.
상대의 이름을 부르지 못함.
```

### 도시 사건 디테일

```text
가로등이 젖은 도로 위에 끊겨 비침.
낡은 철문 아래 먼지가 바깥으로 밀려 있음.
전선이 낮게 처져 있고, 골목은 습기가 많음.
멀리 도시 소음은 있는데 집 앞만 이상하게 조용함.
```

### 판타지/기운 디테일

```text
촛불이 흔들리지 않음.
문틈 안쪽 어둠이 빛을 먹는 듯함.
먼지가 안에서 밖으로 쓸려 나와 있음.
들어오는 빛은 있는데 공기가 움직이지 않음.
동물이 먼저 반응함.
```

### 소품 디테일

```text
닫힌 가방은 바로 열지 않는다.
자료는 필요 시점 전까지 보이지 않는다.
휴대폰/지도/令牌는 다음 질문을 만들 때만 반응시킨다.
핵심 소품은 한 회차에 하나만 강하게 쓴다.
```

## 4. 자동 생성 입력 카드

소설을 만들 때 사용자는 아래만 주면 된다.

```text
회차:
전 회차 마지막 대사:
이번 장면 핵심 사건:
등장인물:
핵심 소품:
장소:
반드시 유지할 대사:
보이면 안 되는 정보:
마지막에 남길 질문:
```

Codex는 위 입력을 바탕으로 자동 생성한다.

```text
캐릭터별 말투
행동 디테일
감정 디테일
장소 디테일
대사 호흡
분경 전 소설 원문
```

## 5. 자동 생성 전 자체 질문

```text
이 디테일은 장면 목적을 돕는가?
이 대사는 이 캐릭터만 할 수 있는가?
이 정보는 지금 공개해도 되는가?
이 행동은 다음 컷으로 이어지는가?
이 장면은 전컷 마지막 대사를 받고 있는가?
```

## 6. 출력 방식

시간 절약을 위해 Codex는 먼저 긴 설명을 하지 말고 아래 형식으로 출력한다.

```text
1. 적용 기준 요약
2. 캐릭터 말투 적용 메모
3. 소설 원문
4. 대사 보존/분할 메모
5. 다음 분경 전 체크
```

단, 사용자가 요청하면 내부 시뮬레이션 전체를 상세 출력한다.
