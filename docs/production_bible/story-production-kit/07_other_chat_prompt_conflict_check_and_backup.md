# Other Chat Prompt: Conflict Check And Backup

아래 문장을 GitHub 저장소를 다루는 다른 Codex 채팅창에 그대로 붙여넣는다.

```text
이 GitHub 저장소는 AI 숏폼 드라마 제작의 공식 원본 기준으로 사용한다.
지금부터 새 작업을 하기 전에, 저장소 안의 기존 세계관/캐릭터/분경/영상 프롬프트/계약/규제 파일을 꼼꼼히 점검해줘.

목표:
1. 기존 작업과 새로 추가한 영상 스타일 분석 파일이 충돌하지 않는지 확인한다.
2. 이전에 만들어진 잘못된 규칙, 과도한 규칙, 악성/오염된 규칙, 서로 모순되는 규칙이 있는지 확인한다.
3. 계약/규제/절대 금지 규칙을 침범하는 새 규칙이 없는지 확인한다.
4. 세계관과 캐릭터 설정이 새 영상 스타일 분석 때문에 바뀌지 않았는지 확인한다.
5. 소설, 분경, 영상 프롬프트 저장 위치가 역할별로 분리되어 있는지 확인한다.

우선순위:
1. 계약/규제/절대 금지 규칙
2. 세계관 설정
3. 캐릭터 설정
4. 장소/소품 설정
5. 기존 분경 분할 규칙
6. 기존 이미지/영상 프롬프트 규칙
7. 새로 추가한 영상 스타일 분석 규칙
8. 회차별 임시 지시

새 영상 스타일 분석 파일의 역할:
- 초반 3~5초 훅 강화
- 대사 명확성 강화
- 인물 진입 타이밍 조정
- 감성 장면 처리
- 판타지 효과와 사건 원인/결과 연결
- 다음화 클리프행어 강화

새 영상 스타일 분석 파일이 하면 안 되는 일:
- 세계관 변경
- 캐릭터 성격/외형 변경
- 계약/규제 파일보다 우선 적용
- 기존 분경 분할 규칙 대체
- 기존 프롬프트 규칙 전체 재작성

점검할 파일/폴더:
1. docs/production_bible/ 또는 유사한 기준 폴더
2. 00_read_order.md
3. 세계관 파일
   관련 브랜치/작업명: codex/create-worldbuilding-markdown-file
4. 캐릭터 설정 파일
   관련 브랜치/작업명: codex/create-character-settings-markdown-file-5zxmil
5. 분경 분할 규칙 파일
6. 이미지/영상 프롬프트 규칙 파일
7. 새로 추가된 영상 스타일 분석 파일
8. episodes/epXX/novel/
9. episodes/epXX/storyboard/
10. episodes/epXX/prompts/

출력 형식:

## 1. 현재 저장소 구조 요약
- 기준 문서 위치:
- 세계관 위치:
- 캐릭터 설정 위치:
- 소설 위치:
- 분경 위치:
- 영상 프롬프트 위치:

## 2. 충돌 없음 / 있음 판단
- 바로 작업 가능 여부:
- 이유:

## 3. 충돌 가능성 목록
- 파일:
- 문제:
- 우선 적용할 규칙:
- 수정 제안:

## 4. 악성/오염/과잉 규칙 점검
- 의심 규칙:
- 왜 위험한지:
- 삭제/격리/하향 조정 여부:

## 5. 최종 read_order.md 제안
- Codex가 읽어야 할 순서:
- 각 파일의 역할:

## 6. 다음 작업 가능 여부
- 소설 작성 가능:
- 분경 작성 가능:
- 영상 프롬프트 작성 가능:
- 추가 수정 필요:

중요:
기존 파일을 바로 덮어쓰지 말고, 먼저 충돌표를 만들어줘.
문제가 없을 때만 새 스타일 분석 파일을 보조 레이어로 확정해줘.
```

## 짧은 버전

```text
저장소의 기존 계약/규제/세계관/캐릭터/분경/프롬프트 규칙을 먼저 점검해.
새로 추가한 영상 스타일 분석은 보조 스타일 레이어로만 적용하고,
기존 설정이나 계약 규칙을 침범하지 않는지 충돌표를 만들어줘.
문제가 없으면 read_order.md 기준으로 소설/분경/프롬프트 작업 가능 여부를 판단해줘.
```
