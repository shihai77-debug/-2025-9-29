# Director Actor Camera Storyboard Execution Gate

목적: 소설이 분경, 카메라, 배우 연기, 영상 프롬프트, CapCut 편집까지 자연스럽게 이어지도록 감독식 실행 기준을 고정한다.

핵심:

```text
좋은 소설 장면은 카메라가 어디에 있어야 하는지 보인다.
좋은 분경은 배우가 무엇을 숨기고 무엇을 드러내야 하는지 보인다.
좋은 영상 프롬프트는 편집점과 다음 컷 연결이 보인다.
```

## 1. Director Question

각 CUT마다 감독 질문을 먼저 답한다.

```markdown
| Question | Answer |
|---|---|
| 이 컷의 관객 시선 1순위는? |  |
| 이 컷의 감정 1순위는? |  |
| 이 컷에서 새로 알게 되는 정보는? |  |
| 카메라는 왜 이 위치에 있는가? |  |
| 이 컷의 마지막 프레임은 다음 컷을 어떻게 부르는가? |  |
```

카메라 위치에 이유가 없으면 그 컷은 약하다.

## 2. Actor Inner / Outer Layer

배우처럼 쓰기 위해 인물마다 겉 행동과 속 감정을 나눈다.

```markdown
| Character | Outer Action | Inner Pressure | Micro Acting | Line Subtext |
|---|---|---|---|---|
| 陆沉舟 |  |  |  |  |
| 黑七 |  |  |  |  |
| 檀缺 |  |  |  |  |
| 沈泊川 |  |  |  |  |
```

원칙:

```text
겉 행동은 촬영 가능해야 한다.
속 감정은 직접 말하지 않는다.
Micro Acting으로 숨긴다.
대사는 Subtext를 가진다.
```

## 3. Camera Angle Function

카메라 각도는 멋으로 고르지 않는다. 기능으로 고른다.

```markdown
| Camera | Use When | Effect |
|---|---|---|
| Static wide | 공간 관계와 긴장 고정 | 인물들이 갇힌 느낌 |
| Low close on hand/door | 위험한 접촉 직전 | 관객이 만지지 말라고 느낀다 |
| Over-shoulder | 한 인물이 다른 인물의 판단을 기다림 | 관계와 압박 |
| Medium close | 인물의 판단/숨김 감정 | 배우 얼굴과 손 동작 |
| Insert | 소품/먼지/문틈/符痕 단서 | 설명 대신 정보 |
| Slow push-in | 깨달음 또는 위험이 다가옴 | 관객 집중 |
| Cutaway reaction | 긴 대사 호흡 분할 | 설명이 덜 지루함 |
| Faceless master | 3인 이상 위치 확인 | 얼굴 붕괴 방지 |
```

금지:

```text
모든 중요한 대사를 같은 정면샷으로 처리
카메라 움직임이 감정/정보 변화 없이 예쁘기만 함
인서트 없이 말로만 설명
3인 이상 얼굴을 오래 선명하게 보여주며 긴 대사
```

## 4. Shot-To-Story Link

각 컷은 이야기 기능을 가져야 한다.

```markdown
| CUT | Story Function | Camera Function | Actor Function | Edit Anchor |
|---|---|---|---|---|
| 21-01 | 20화 끝 위험 받기 | 문/손/숨소리 인서트 | SHEN 멈춤, LU 시선 고정 | 숨소리로 다음 컷 |
| 21-02 | 귀신 오해 깨기 | 黑七 낮은 위치 리액션 | 감각+불쾌감 | 문틈으로 이동 |
```

이 표 없이 분경으로 넘어가지 않는다.

## 5. Dialogue Blocking

대사는 입만 움직이면 안 된다.

대사마다 아래를 붙인다.

```text
누가 말하는가
말하기 전 무엇을 봤는가
말하는 동안 손/시선/몸은 무엇을 하는가
상대는 어떻게 듣는가
말 뒤에 무엇이 변하는가
```

변화가 없으면 대사를 삭제하거나 행동으로 바꾼다.

## 6. Cut Ending Rule

각 CUT의 마지막 프레임은 다음 컷의 이유가 되어야 한다.

좋은 연결:

```text
문고리 떨림 -> 손 클로즈업
黑七 귀 눕힘 -> 문틈 소리
沈泊川 질문 -> LU가 대답 대신 바닥을 봄
먼지 이동 -> 符痕 인서트
LU 침묵 -> 다음 컷에서 낮은 판단
```

나쁜 연결:

```text
장면이 그냥 끝남
인물이 말하고 정지
풍경으로 도망
다음 컷이 새로 시작함
```

## 7. Famous Director Simulation

출력 전에 감독처럼 세 번 본다.

```text
1차: 무음으로 봐도 사건이 이해되는가?
2차: 화면 없이 대사만 들어도 캐릭터가 구분되는가?
3차: 편집표로 봤을 때 컷마다 새 정보/감정/위험이 있는가?
```

셋 중 하나라도 실패하면 다시 쓴다.

## 8. Novelist Simulation

소설가처럼 한 번 더 본다.

```text
문장이 예쁜가보다 먼저, 장면이 필요한가?
심리 설명보다 행동 디테일이 있는가?
장면마다 인물 선택이 있는가?
마지막 문장이 다음 문단을 부르는가?
캐릭터가 작가 대신 설명하고 있지 않은가?
```

## 9. 21화 카메라 예시

21화 老宅 첫 장면 추천 흐름:

```text
01. Insert close: 문고리 앞에서 멈춘 손, 안쪽 숨소리.
02. Static medium: 네 사람 위치, 누구도 바로 움직이지 않음.
03. Low angle: 黑七 귀와 꼬리의 낮은 반응, 문틈 방향.
04. Insert: 먼지가 문 밖으로 나와야 하는데 안쪽으로 빨림.
05. Medium close: 陆沉舟가 문이 아니라 바닥/문틀을 본다.
06. Over-shoulder: 沈泊川가 LU 판단을 기다림.
07. Close insert: 符痕 또는 비정상적으로 깨끗한 문틈.
08. Reaction chain: 黑七 -> 檀缺 -> SHEN -> LU.
09. Final frame: 문 안쪽에서 새 소리/흔적이 반응.
```

이 흐름은 그대로 복사하라는 뜻이 아니다. 기능을 따라야 한다.

## 10. Final Command

분경/영상 프롬프트 전 아래 명령을 실행한다.

```text
Director Actor Camera Gate를 실행해.
각 CUT마다 Story Function, Camera Function, Actor Inner/Outer Layer, Edit Anchor를 표로 만든 뒤,
기능 없는 컷은 삭제하거나 합쳐.
```
