# Self Updating Production Loop

목적: 소설, 분경, 영상 프롬프트, 실제 영상 결과를 계속 개선하기 위한 자가업데이트형 제작 시스템을 만든다.

중요:

```text
자가업데이트 = AI가 마음대로 설정을 바꾸는 것 아님.
자가업데이트 = 매 회차 결과를 검수하고, 검증된 개선만 제작 규칙에 반영하는 것.
```

## 1. 공식 원칙

```text
세계관/캐릭터는 쉽게 바꾸지 않는다.
스타일/분경/프롬프트 규칙은 결과에 따라 업데이트한다.
영상 결과가 좋았던 이유와 실패한 이유를 반드시 기록한다.
규칙 업데이트는 "증거"가 있을 때만 한다.
```

## 3. 매 회차 제작 순서

```text
1. read_order, 세계관, 캐릭터 카드 확인
2. 전 회차/전 컷 마지막 대사와 마지막 행동 확인
3. 소설 원문 작성
4. 장면 질문 확인
5. 대사 보존 목록 작성
6. 비트 분할
7. CUT 분경 작성
8. 영상 프롬프트 작성
9. 저비용 테스트 영상 생성
10. 결과 검수
11. 좋은 점/실패점 기록
12. 규칙 업데이트 후보 작성
13. 검증된 규칙만 production_bible에 반영
```

## 4. 매 회차 검수표

```text
회차:
장면:
첫 3초 훅:
시청자가 갖는 질문:
핵심 소품:
핵심 대사:
감정 중심:
판타지 요소:
다음화 유도:
이해 안 되는 부분:
끌림이 약한 부분:
재생성 필요 여부:
규칙 업데이트 후보:
```

## 5. 업데이트 판단 기준

업데이트해도 되는 것:

```text
초반 훅 방식
대사 길이 기준
분경 컷 길이
인물 진입 타이밍
소품 공개 타이밍
다음화 클리프행어 방식
영상 프롬프트 실패 방지 문장
```

업데이트하면 안 되는 것:

```text
캐릭터 얼굴/나이/성격
세계관 핵심 설정
이미 확정된 관계
계약/규제/금지사항
회차별 이미 공개된 사실
```

## 6. 성공/실패 기록 방식

좋은 결과는 왜 좋았는지 기록한다.

```text
성공한 컷:
성공 이유:
재사용 가능한 규칙:
다음 회차 적용 위치:
```

실패한 결과도 반드시 기록한다.

```text
실패한 컷:
실패 유형:
원인:
다음 프롬프트 수정:
규칙 업데이트 필요 여부:
```

실패 유형:

```text
캐릭터 붕괴
장소 불연속
소품 누락
자료 너무 빨리 공개
대사 의미 변경
초반 훅 약함
다음화 유도 약함
판타지 효과만 있고 대가 없음
인물 너무 많이 등장
```

## 7. A/B 테스트 운영

본편 전체를 여러 번 만들지 않는다.

테스트할 것:

```text
도입부 3초
첫 대사
마지막 컷
다음화 예고 질문
```

도입부 A/B/C 예시:

```text
A: 정적과 얼굴로 시작
B: 이상한 소리/오토바이 엔진으로 시작
C: 닫힌 소품이 먼저 반응하는 것으로 시작
```

선택 기준:

```text
가장 빨리 질문을 만드는 버전
가장 적게 설명해도 이해되는 버전
다음 컷을 보고 싶게 만드는 버전
```

## 8. 15~20화 현재 흐름 점검

### 15화 — 檀骨归位

기능:

```text
檀缺 생존 확정
천 년의 무게를 감정으로 고정
산에서 내려오며 큰 장 종료
```

좋은 대사:

```text
"才一千年。"
"活着就是重的。"
```

역할:

```text
감성 회차
생존과 귀환의 무게
다음 마을/도시 생활로 넘어가는 완충
```

### 16화 — 出棺第一天

기능:

```text
마을 첫날
흰 옷 문제로 가볍게 시작
옷가게에서 遮气阵 흔적 발견
```

좋은 점:

```text
일상 코미디와 사건 단서가 같이 있음
마네킹에게 "辛苦了。"는 기억에 남는 이상한 감성
```

강화 방향:

```text
遮气阵 흔적을 단순 단서가 아니라 다음 냄새/추적과 연결한다.
```

### 17화 — 气息

기능:

```text
黑七의 추적 능력 강조
동물 네트워크 활용
천 년 동안 처음 맡는 냄새 발견
```

좋은 대사:

```text
"活了一千年。就这一次，说不上来。"
```

강화 방향:

```text
냄새의 정체를 바로 설명하지 말고, 다음 회차의 인물/인장/과거와 연결한다.
```

### 18화 — 女王印

기능:

```text
장군 등장
陆沉舟와 충돌
무릎 꿇음
女王印 전달
白清檀 기억 완전 회복
```

좋은 대사:

```text
"末将……找了陛下千年。您还活着。末将就知足了。"
```

역할:

```text
감정과 신분 폭로 회차
백청단이 단순 동행자가 아니라 여왕 축으로 올라감
```

### 19화 — 昆仑

기능:

```text
白清檀이 昆仑山으로 떠남
陆沉舟가 결계에 세 번 부딪힘
昆仑의 규칙 제시
3개월 약속
```

좋은 대사:

```text
"那我等你。"
"等我回来。三个月。昆仑。"
```

역할:

```text
큰 감정 절단
20화의 죽은 공기를 만드는 원인
```

### 20화 — 첫 도시 의뢰 시작

기능:

```text
昆仑 이후 정적 보존
沈泊川 오토바이로 외부 사건 진입
韩知玄 镇灵司 복귀
城南 老宅 의뢰 시작
陆沉舟가 一看形으로 局 발견
```

좋은 점:

```text
19화 감정에서 도시 사건으로 넘어가는 연결이 명확하다.
沈泊川는 분위기를 읽지만 캐묻지 않아 좋다.
韩知玄 퇴장은 기능적이고 자연스럽다.
老宅의 이상함은 "귀신이 아니라 사람이 막은 기운"이라 차별점이 있다.
```

주의점:

```text
CUT-01에서는 자료를 보이면 안 된다.
자료 공개는 CUT-02 이후 委托入局에서 해야 한다.
20화 끝은 "어떻게 하지?"보다 더 강한 질문이 있으면 좋다.
```

20화 끝 강화 후보:

```text
陆沉舟가 문 앞에서 멈춘 순간, 문 안쪽에서 누군가 안에서 문을 세 번 두드린다.
黑七이 "里面没有活人。"이라고 말하는데, 바로 안쪽에서 사람 숨소리가 들린다.
沈泊川의 닫힌 背包 안에서 꺼져 있던 휴대폰이 켜지고, 발신자 이름이 白清檀으로 뜬다.
문 아래 쓸려 나온 먼지 속에서 女王印과 같은 문양이 드러난다.
陆沉舟가 "这里不是闹鬼。"라고 말한 직후, 문 안쪽에서 "陆沉舟"라는 목소리가 들린다.
```

추천:

```text
20화는 현재 구조 그대로 사용 가능.
단, 마지막 5초에 "다음화 질문"을 하나 더 강하게 넣으면 수익형 숏폼 구조에 더 맞다.
```

## 9. 현재 20화 대사 점검

좋은 대사:

```text
"看你们几个这脸色，跟天塌了半截似的。"
"我今天不是来探你们私事的，有个棘手的委托。"
"这事不对劲，必须得有人去看看。"
"别动。"
"这里不是闹鬼。"
```

줄이면 좋은 대사:

```text
"城南老城区一栋老旧民宅，接连几个保洁工人进去打扫，全都莫名窒息心慌，拼了命从屋里冲出来，没人敢再踏进一步。"
```

분경용 제안:

```text
"城南老宅，进去打扫的人，全都喘不上气。"
"不是撞鬼。更像有人把整栋屋子的气堵死了。"
```

이유:

```text
영상에서는 긴 정보 대사가 약하다.
자료 화면, 문 앞 먼지, 인물 반응으로 나눠 보여주는 편이 좋다.
```

보강:

```text
대사를 무조건 짧게 만들면 안 된다.
캐릭터 표현력과 감정은 유지해야 한다.
긴 설명 대사는 삭제하지 말고, 대사/자료/반응 컷으로 나눈다.
```

전컷 연결:

```text
19화의 "那我等你。" / "等我回来。三个月。昆仑。"이 20화 첫 정적의 이유다.
20화 첫 컷은 새로운 사건이 아니라 기다림의 정지 상태에서 시작해야 한다.
沈泊川의 오토바이 소리는 그 기다림을 끊는 외부 현실의 침입이다.
```

## 10. 자가업데이트 실행 명령문

다음부터 Codex에 이렇게 지시한다.

```text
GitHub 저장소의 production_bible/read_order 기준을 먼저 읽어.
이번 회차 소설/분경/프롬프트를 만든 뒤,
반드시 review 폴더에 아래 3개를 작성해.

1. story_logic_check
2. prompt_result_review
3. update_candidates

단, update_candidates는 바로 production_bible에 반영하지 말고,
사용자 확인 후 검증된 규칙만 반영해.
```

## 11. 감독/배우/카메라 시뮬레이션 필수화

소설, 분경, 영상 프롬프트를 만들기 전에 아래를 반드시 수행한다.

```text
1. 전컷 마지막 대사/행동 연결 확인
2. 캐릭터별 배우 동기 설정
3. 대사 호흡 검수
4. 인물 위치와 동선 설계
5. 카메라 위치와 컷 의도 설계
6. 편집 리듬 시뮬레이션
7. 마지막 클리프행어 강도 확인
```

자세한 기준:

```text
outputs/story-production-kit/10_director_actor_camera_simulation_workflow.md
```

자동화 원칙:

```text
사용자 피드백이 없어도 기본 검수는 Codex가 먼저 수행한다.
하지만 세계관/캐릭터/관계/정체 공개 같은 핵심 설정 변경은 자동 확정하지 않는다.
```

## 12. 말투/디테일 자동 생성

시간 절약을 위해 말투와 디테일은 별도 시스템으로 자동 적용한다.

```text
캐릭터 말투는 캐릭터 카드와 voice sheet 기준으로 자동 적용한다.
행동/감정/장소/소품 디테일은 장면 목적 안에서 자동 생성한다.
자동 생성된 디테일은 세계관과 캐릭터 설정을 바꾸면 안 된다.
```

관련 문서:

운영 원칙:

