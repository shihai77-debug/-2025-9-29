# Dialogue Character Emotion Story Rules

목적: 대사를 단순 정보 전달이 아니라 재미, 캐릭터성, 감정, 스토리 추진력을 동시에 가진 장면 엔진으로 만든다.

핵심:

```text
대사는 짧기만 하면 안 된다.
대사는 정보만 말해도 안 된다.
좋은 대사는 캐릭터 성격 + 현재 감정 + 스토리 정보/단서를 동시에 담는다.
하지만 그 전에, 인물이 왜 지금 말할 수밖에 없는지부터 정해야 한다.
```

## 0. 대사 전 무대 필연성

대사는 인물 기능표에서 나오면 안 된다. 반드시 현장 압력에서 나와야 한다.

```text
1. 지금 장면에서 누가 가장 위험한가?
2. 누가 먼저 말할 수밖에 없는가?
3. 누가 말하지 않고 행동해야 하는가?
4. 이 대사가 없으면 장면이 안 움직이는가?
```

이 네 질문을 통과하지 못한 대사는 삭제한다.

금지:

```text
沈泊川 질문 -> 黑七 감각 -> 檀缺 찌르기 -> 陆沉舟 정리
```

이 순서는 장면이 실제로 요구할 때만 쓴다. 기본값으로 쓰면 캐릭터가 아니라 역할표가 된다.

## 1. 대사 3중 포함 원칙

모든 주요 대사는 아래 3가지를 동시에 가져야 한다.

```text
1. 캐릭터 성격
2. 현재 감정
3. 스토리 정보 / 단서 / 반전 / 긴장
```

나쁜 대사:

```text
이 집 위험하다.
```

좋은 대사:

```text
沈泊川: "这宅子，我一开始还当普通委托接的。现在想想，接得有点太轻了。"
```

이 대사의 기능:

```text
성격: 현실적이고 겁도 있음
감정: 불안, 후회
정보: 집이 보통 의뢰가 아님
```

## 2. 인물별 말투 온도차

티키타카는 인물 말투 온도차에서 나온다.

### LU / 陆沉舟

```text
판단 먼저, 설명 나중.
길게 감정 설명하지 않음.
말을 줄여 멋있어 보이려 하지 않는다.
필요할 때는 한 문장보다 길게 말해도 된다. 단, 말마다 판단/정보/감정 압력이 있어야 한다.
무표정한 판단 속에 숨은 긴장을 둔다.
```

대사 기능:

```text
제지
판단
구조 분석
결론
위험 경고
현장 장악
다음 행동 지시
```

예시:

```text
"手别碰门。它不像锁，更像等人替它开口。"
"有鬼的地方不会这么干净。这里被人洗过气。"
"这是局，但不是吓人的局，是关人的局。"
"门一开，先出来的不一定是人，推门的人也未必退得回来。"
```

### HEIQI / 黑七

```text
선 비꼼, 후 걱정.
짧고 리액션이 빠르다.
상황을 가장 먼저 감지하지만 처음엔 가볍게 말한다.
진짜 위험할 때만 톤이 낮아진다.
```

대사 기능:

```text
감각 반응
비꼼
본능 경고
위기 순간의 진지한 톤 전환
```

예시:

```text
"嚯，这屋子还挺会装死。"
"没有鬼味。"
"等等。这个味儿……不对。"
"陆沉舟，别让他开门。"
```

### TAN / 檀缺

```text
흑칠 저격.
한마디가 짧고 쨍하다.
관찰자 시점에서 팩트만 찌른다.
말수는 적지만 캐릭터 대비를 만든다.
```

대사 기능:

```text
흑칠 견제
상황 정리
차가운 관찰
짧은 팩트 폭로
```

예시:

```text
"你又闻出什么废话了？"
"他说得对。"
"不是怕，是这地方不让人喘。"
```

### SHEN / 沈泊川

```text
시청자 대변인.
불안, 의문, 당황을 보여준다.
현실적인 말투로 관객이 따라오게 한다.
가볍게 들어오지만 책임감 있게 본론을 전달한다.
```

대사 기능:

```text
상황 질문
현실 반응
의뢰 정보
공포 체감
관객 대신 의문 제기
```

예시:

```text
"我怎么听着不像凶宅，倒像有人在里面憋着一整栋楼的气？"
"你们能不能先告诉我，这门到底是不能开，还是开了就没命？"
```

## 3. 같은 상황, 다른 반응

재미있는 대사는 같은 사건을 각 인물이 다르게 받는 데서 나온다.

예: 老宅가 사실 사람을 가두는 局이라는 반전.

```text
LU: 구조를 분석한다. 이성.
HEIQI: 냄새와 본능으로 먼저 이상함을 잡는다. 감각.
TAN: 상황을 차갑게 정리한다. 관찰.
SHEN: 사람 입장에서 무섭고 이해 안 되는 반응을 한다. 현실.
```

이 온도차가 장면을 살린다.

## 4. 감정 톤 변화 공식

캐릭터는 평소 말투를 유지하다가, 위기에서만 미세하게 변해야 한다.

```text
평소 톤
→ 상황 긴장 시 미세 변화
→ 클라이맥스에서 톤 전환
```

HEIQI:

```text
평소: 비꼼, 투덜거림
긴장: 농담이 줄어듦
위기: 낮고 진지하게 경고
```

LU:

```text
평소: 낮고 무미건조한 판단
그리움/상실: 말이 더 짧아지고 침묵이 길어짐
위기: 설명보다 제지가 먼저 나옴
```

SHEN:

```text
평소: 현실적인 농담과 빠른 정보 전달
긴장: 말이 빨라짐
위기: 질문이 짧아지고 숨이 섞임
```

TAN:

```text
평소: 짧고 차갑게 찌름
긴장: 말수가 더 줄어듦
위기: 팩트 한마디만 남김
```

## 5. 의문, 반박, 미궁 원칙

대사는 설명보다 질문을 만들어야 한다.

필수 기능:

```text
의문
반박
미궁
충돌
다음 행동 압박
```

나쁜 해결 대사:

```text
"이 집은 술법 때문에 위험하다."
```

좋은 해결 후 미궁:

```text
LU: "这是局。"
SHEN: "困谁的？"
HEIQI: "问题是，里面没有活人。"
문 안쪽에서 낮은 숨소리.
```

해결했지만 더 큰 질문이 남는다.

```text
누가 갇혔는가?
왜 산 사람이 없는데 숨소리가 나는가?
누가 이 局을 만들었는가?
```

## 6. 장면별 티키타카 사용량

티키타카는 모든 장면에 많이 넣으면 안 된다.

```text
일상/이동/조사 초반: 티키타카 허용
정보 정리 장면: 짧은 반박/질문 허용
공포/위기 장면: 티키타카 줄임
감정 장면: 농담 최소화
클리프행어 직전: 말수 줄이고 핵심 한마디만
```

## 7. 대사 설계 절차

대사를 쓰기 전에 아래를 채운다.

```text
장면 사건:
이 대사가 필요한 이유:
발화자:
캐릭터 성격:
현재 감정:
스토리 정보:
숨겨야 할 정보:
다음 컷으로 넘길 질문:
최종 대사:
```

검수 질문:

```text
이 대사는 누가 말해도 되는 말인가?
아니면 이 캐릭터만 할 수 있는 말인가?
정보만 말하고 있지 않은가?
감정만 말하고 있지 않은가?
다음 컷을 열고 있는가?
```

## 8. 20화 적용 예시

상황: 城南 老宅 앞, 문을 열려는 SHEN을 LU가 막는다.

```text
LU: "别动。"
SHEN: "……门都还没碰呢。"
HEIQI: "你再往前半步，就不是碰门，是送命。"
TAN: "他说得难听，但这次没错。"
SHEN: "行。那我现在开始害怕，来得及吗？"
LU: "来得及。别开门就行。"
```

기능:

```text
LU: 제지와 판단
SHEN: 시청자식 현실 반응
HEIQI: 비꼼+걱정
TAN: 흑칠 저격+팩트
스토리 정보: 문을 열면 위험
감정: 긴장 속 캐릭터 온도차
다음 컷: 왜 문을 열면 안 되는지 분석으로 이동
```

## 9. 공식 적용 규칙

앞으로 모든 소설/분경 대사는 아래를 통과해야 한다.

```text
정보+감정+캐릭터 중 최소 2개 이상 포함.
주요 대사는 3개 모두 포함.
네 인물의 온도차 유지.
해결 대사 뒤에는 더 큰 의문을 남김.
위기에서는 톤 변화가 보여야 함.
캐릭터 카드와 충돌하면 캐릭터 카드 우선.
```
