# Fast Episode Input Card

목적: 시간이 부족할 때 사용자가 최소 정보만 주면 Codex가 세계관/캐릭터/스타일 기준을 읽고 소설, 분경, 영상 프롬프트를 이어서 만들 수 있게 한다.

## 1. 사용자 입력 양식

아래만 채우면 된다.

```text
작업 종류:
소설 / 분경 / 영상 프롬프트 / 전체

회차:

전 회차 마지막 대사 또는 마지막 행동:

이번 회차 첫 이미지:

이번 회차 핵심 사건:

등장인물:

핵심 소품:

장소:

반드시 들어갈 대사:

절대 공개하면 안 되는 정보:

마지막에 남길 질문:

원하는 톤:
감성 / 미스터리 / 사건 추진 / 도시 의뢰 / 판타지 / 코미디 섞기
```

## 2. Codex 자동 처리 순서

```text
1. read_order 기준 확인
2. 세계관/캐릭터/소품/장소 확인
3. 전컷 마지막 대사 연결 확인
4. 감독/배우/카메라 시뮬레이션
5. 캐릭터 말투 자동 적용
6. 디테일 자동 생성
7. 소설 원문 작성
8. 분경 분할
9. 영상 프롬프트 작성
10. review/update_candidates 작성
```

## 3. 사용 예시

```text
작업 종류: 소설
회차: 20화
전 회차 마지막 대사: "等我回来。三个月。昆仑。" / "那我等你。"
이번 회차 첫 이미지: 陆沉舟가 昆仑이 사라진 방향을 보고 움직이지 않음
이번 회차 핵심 사건: 沈泊川가 城南老宅 의뢰를 가져옴
등장인물: LU, HEIQI, TAN, HAN, SHEN
핵심 소품: 닫힌 背包, 摩托车, 头盔, 의뢰资料
장소: 昆仑 이후 旷野 → 城南 老宅
반드시 들어갈 대사: "别动。" / "这里不是闹鬼。"
절대 공개하면 안 되는 정보: CUT-01에서 资料 내용 노출 금지
마지막에 남길 질문: 老宅 안에 귀신은 없는데 왜 숨소리가 들리는가
원하는 톤: 감성에서 도시 미스터리로 직진
```

## 4. 시간 절약 운영법

사용자는 매번 긴 설명을 하지 않아도 된다.

```text
Fast Episode Input Card만 작성
Codex가 production_bible을 읽음
Codex가 말투/디테일/카메라/분경을 자동 보강
사용자는 결과만 보고 큰 방향 피드백
```
