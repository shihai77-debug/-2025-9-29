# Bad Example Quarantine And Style Do Not Copy

목적: 문서 안의 나쁜 예시, 과거 실패 대사, 반복된 안전 문장을 AI가 그대로 따라 쓰는 문제를 막는다.

핵심:

```text
문서에 많이 나온 문장 = 써도 되는 문장이 아니다.
나쁜 예시로 나온 문장 = 절대 복사 금지다.
규칙 문서는 창작 재료가 아니라 검사 기준이다.
```

## 1. 격리 대상 문장

아래 문장/구조는 특별히 허락하지 않는 한 새 소설/대사에 그대로 쓰지 않는다.

```text
别动。
不是鬼。
不是人。
那怎么办？
这也太怪了吧。
好厉害。
里面有人？
这里不是闹鬼。
我查不到源头。
```

이 문장들은 과거 구조 설명, 실패 예시, 압축 예시로 반복 등장했기 때문에 AI가 쉽게 베낀다.

## 2. 사용 가능 조건

위 문장을 꼭 써야 할 때는 아래 조건을 모두 만족해야 한다.

```text
원문 source에 이미 확정된 대사다.
캐릭터 카드와 회차 감정에 정확히 맞다.
대사 앞뒤 행동/반응/소품이 새 정보를 만든다.
반복 사용이 아니라 이번 장면에서만 필요한 기능이 있다.
```

하나라도 부족하면 새 표현으로 바꾼다.

## 3. 나쁜 예시를 읽는 방식

문서 안의 나쁜 예시는 이렇게 읽는다.

```text
"이렇게 쓰지 말라"는 신호
"이 기능을 다른 말로 바꾸라"는 신호
"대사보다 행동/소품/반응으로 나누라"는 신호
```

나쁜 예시는 절대 아래처럼 쓰지 않는다.

```text
캐릭터 말투 참고
최종 대사 후보
프롬프트 문장
자막 원문
```

## 4. 창작 전 변환 규칙

나쁜 예시와 같은 기능이 필요하면 먼저 기능으로 바꾼다.

```text
别动。 -> 제지 / 위험 차단 / 문고리 접촉 금지
不是鬼。 -> 오해 깨기 / 일반 공포물 방향 부정
那怎么办？ -> 현실적 선택 압박 / 관객 질문 대변
这里不是闹鬼。 -> 사건 장르 전환 / 局 가능성 제시
```

그다음 캐릭터별 새 표현으로 다시 쓴다.

예:

```text
LU 제지:
手离门远点。
退半步。别踩线。
它等的就是有人先碰。

SHEN 현실 질문:
我刚才要是真推了，是不是就不用问后面了？
你先告诉我，我现在站的位置安全吗？
这单我是不是接得太便宜了？

HEIQI 감각:
这味儿不脏，干净得像被洗过命。
有东西把气咬断了。
门缝里没有鬼味，可我不想靠近。

TAN 관찰:
不是没人住，是气不准出来。
门没锁，锁的是里面那口气。
```

## 5. 출력 전 중복 검사

최종 소설/분경/대사 전에 반드시 검사한다.

```text
금지 문장이 그대로 들어갔는가?
나쁜 예시의 문장 구조를 그대로 바꿔 썼는가?
같은 기능의 대사가 두 번 이상 반복되는가?
캐릭터가 자기 말투가 아니라 규칙 문서 말투로 말하는가?
대사가 장면을 전진시키지 않고 안전한 설명만 하는가?
```

있으면 삭제/수정한다.

## 6. GitHub 작업자용 명령

다른 GitHub/Codex 채팅창이 대사를 못 쓰거나 반복할 때 아래 문장을 보낸다.

```text
30_bad_example_quarantine_and_style_do_not_copy.md를 실행해.
문서에 반복 등장한 나쁜 예시 대사를 최종 대사로 쓰지 말고,
대사 기능만 추출해서 캐릭터별 새 표현으로 다시 써.
```

## 7. 최종 원칙

```text
규칙을 베끼지 말고, 규칙으로 검사한다.
예시를 베끼지 말고, 기능을 훔친 뒤 새 문장으로 만든다.
```
