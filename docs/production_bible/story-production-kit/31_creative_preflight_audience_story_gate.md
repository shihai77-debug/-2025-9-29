# Creative Preflight Audience Story Gate

목적: 소설을 쓰기 전에 예술적 판단, 관객 흡입력, 인물 존재 이유, 뒤 회차 연결성을 먼저 검증한다. 이 문서는 창작 전 필수 통과 게이트다.

핵심:

```text
첫 장면이 약하면 뒤 분경, 영상, 편집이 모두 힘들어진다.
첫 단추는 문장보다 구조다.
구조보다 먼저 관객 질문이다.
관객 질문보다 먼저 인물의 이유다.
```

## 1. 쓰기 전 금지

아래 상태에서는 소설을 시작하지 않는다.

```text
첫 3초 질문이 없음
왜 이 인물이 여기 있는지 불명확함
장면이 끝날 때 상태 변화가 없음
관객이 무엇을 기대할지 예측하지 않음
뒤 분경/영상/편집에서 무엇을 찍을지 안 보임
대사가 인물의 선택이 아니라 설명으로만 존재함
```

## 2. First Button Gate

모든 회차/장면은 아래 표를 먼저 채운다.

```markdown
| Item | Answer |
|---|---|
| 이전 화 마지막 이미지/대사/소리 |  |
| 이번 화 첫 이미지 |  |
| 첫 3초 관객 질문 |  |
| 8초 안에 바뀌는 상태 |  |
| 20초 안에 깨지는 오해 |  |
| 장면 끝 새 질문 |  |
| 다음 장면으로 넘길 물리적 연결 |  |
```

통과 기준:

```text
첫 이미지가 이전 화 마지막을 받아야 한다.
첫 3초 질문은 설명이 아니라 화면/소리/행동으로 생겨야 한다.
장면 끝 질문은 시작 질문과 달라야 한다.
```

## 3. Audience Retention Ladder

관객 시점에서 계속 볼 이유를 시간대별로 설계한다.

```markdown
| Time | Viewer Question | Why They Keep Watching |
|---|---|---|
| 0-3s |  |  |
| 3-8s |  |  |
| 8-20s |  |  |
| 20-40s |  |  |
| Final 5s |  |  |
```

실패:

```text
0-3초가 분위기뿐이다.
8초까지 사건 반응이 없다.
20초까지 새 정보가 없다.
40초까지 선택/위험/반전이 없다.
마지막 질문이 약하다.
```

## 4. Character Why-Here Gate

등장인물마다 반드시 이유가 있어야 한다.

```markdown
| Character | Why Here Now | Wants | Pressure | What Only They Can Do | If Removed, What Breaks? |
|---|---|---|---|---|---|
| 陆沉舟 |  |  |  |  |  |
| 黑七 |  |  |  |  |  |
| 檀缺 |  |  |  |  |  |
| 沈泊川 |  |  |  |  |  |
| 韩知玄 |  |  |  |  |  |
```

삭제 기준:

```text
인물을 빼도 장면이 그대로면 그 인물은 말하지 않는다.
인물을 빼도 장면이 더 선명하면 그 인물은 다음 장면으로 미룬다.
인물이 있어야 하는 이유가 직무/감각/관계/소품 중 하나로 설명되어야 한다.
```

## 5. Character Choice Gate

인물은 설명하는 존재가 아니라 선택하는 존재다.

각 주요 인물은 장면에서 최소 하나를 가져야 한다.

```text
멈춘다
막는다
숨긴다
다가간다
물러난다
묻는다
반박한다
확인한다
거짓 오해를 깨뜨린다
다음 행동을 정한다
```

대사가 있어도 선택이 없으면 약하다.

## 6. Viewer Prediction And Subversion

관객은 항상 다음을 예측한다. 작가는 그 예측을 알아야 한다.

```markdown
| Viewer Expects | Story Actually Does | Foreshadow Needed |
|---|---|---|
| 凶宅에는 귀신이 있다 | 귀신이 아니라 누군가가 气를 막았다 | 먼지, 바람 없음, 黑七의 감각 |
| 문을 열면 사건이 시작된다 | 문을 열면 이미 늦는다 | 문고리 반응, 안쪽 숨소리 |
```

반전 조건:

```text
관객 예측을 알아야 반전이 가능하다.
복선 없는 반전은 금지다.
예측을 깨도 세계관 규칙은 지켜야 한다.
```

## 7. Emotional Motor

장면의 감정 엔진을 한 문장으로 정한다.

```text
이 장면의 겉 사건:
이 장면의 속 감정:
주인공이 숨기는 감정:
관객이 느껴야 할 감정:
```

예:

```text
겉 사건: 老宅 문 앞에서 局을 확인한다.
속 감정: 陆沉舟는 昆仑의 기다림에서 현실 사건으로 억지로 돌아온다.
숨기는 감정: 아직 白清檀 쪽에 마음이 남아 있다.
관객 감정: 슬픔이 끊긴 자리에 더 이상한 위험이 들어왔다.
```

## 8. Downstream Continuity Gate

소설을 쓰기 전에 뒤 작업까지 생각한다.

```markdown
| Later Stage | Required Anchor |
|---|---|
| Storyboard | 첫 이미지, 인물 위치, 핵심 소품, 마지막 질문 |
| Image Prompt | 고정 캐릭터, 장소, 조명, 금지 요소 |
| Video Prompt | 움직임, 대사 타이밍, 카메라, 소리 |
| CapCut | 컷 길이, 자막, SFX, 음악, 전환 |
| Next Episode | 남겨야 할 물리적/감정적 연결 |
```

아래가 없으면 소설을 확정하지 않는다.

```text
분경 첫 컷으로 보이는 이미지
영상 프롬프트로 움직일 수 있는 행동
CapCut에서 붙일 수 있는 소리/전환
다음 장면으로 넘길 물리적 anchor
```

## 9. 21화 적용 예시

21화는 새 출발이 아니다.

```text
이전 화 마지막: 老宅 문 앞, 열면 안 되는 위험, 안쪽 반응.
첫 이미지: 문고리에 닿기 직전 멈춘 손 또는 문틈 안쪽의 두 번째 숨.
첫 질문: 안에 사람이 없다면, 누가 숨을 쉬는가?
8초 변화: 黑七의 감각 또는 沈泊川의 손이 멈춤.
20초 오해 깨기: 귀신 문제가 아니라 气의 흐름이 막힌 문제.
40초 위험: 문을 열면 안쪽에 갇힌 것이 출구를 얻는다.
마지막 질문: 안쪽의 흔적이 白清檀/无相会/과거 사건과 연결되는가?
```

## 10. Final Preflight Command

작업 전 아래 명령을 반드시 수행한다.

```text
Creative Preflight를 먼저 실행해.
First Button Gate, Audience Retention Ladder, Character Why-Here Gate, Viewer Prediction 표를 작성하고,
통과하지 못하면 소설을 쓰지 말고 구조부터 고쳐.
```
