# 지멍/Jimeng 영상 프롬프트용 대사 읽기 기준

용도: 다음에 사용자가 “영상용 프롬 써줘”, “지멍 영상 프롬프트”, “대사 자연스럽게 읽히게”라고 요청할 때 참고한다.  
목표: AI가 대사를 문학 설명문처럼 읽지 않고, 실제 사람이 말하는 톤으로 읽도록 프롬프트 안에 대사/감정/호흡/속도를 분리해 넣는다.

---

## 1. 핵심 원칙

```text
대사 = 실제 입으로 말할 문장만 쓴다.
감정/연기/상태 = 대사 밖에 따로 쓴다.
호흡/멈춤 = 구두점과 짧은 지시로 표시한다.
한 줄 대사 = 너무 길게 쓰지 않는다.
문학 설명을 대사 안에 넣지 않는다.
```

나쁜 방식:

```text
陆沉舟冷冷地看着门框，眼神复杂又带着压迫感，声音低沉而危险地说：“手别靠近。停住。”
```

좋은 방식:

```text
角色：陆沉舟
语气：低声，稳，压住情绪，不喊
节奏：慢半拍，句尾压低
状态：视线锁住铁门，身体不动
台词：手别靠近。停住。
```

---

## 2. 대사 입력 구조

영상 프롬프트 안에서 대사는 아래 구조로 작성한다.

```text
【语音提示】
角色：角色名
语气：低声 / 紧张 / 平静 / 压着说 / 不喊
节奏：慢半拍 / 短促 / 前快后慢 / 中间停顿
状态：视线、手、身体、呼吸等 화면에 보이는 상태

【台词】
角色真正说出口的话。
```

간단형:

```text
陆沉舟｜低声，稳，不喊，句尾压低：
手别靠近。停住。

沈泊川｜紧张，呼吸发紧，前半句快后半句慢：
我没碰。只是没想到，空宅里还能有动静。
```

---

## 3. 구두점으로 호흡 제어

```text
， = 짧은 쉼
。 = 확실한 멈춤
…… = 망설임 / 숨 멈춤 / 말끝 삼킴 / 여운
？ = 의심 / 반문
！ = 강한 감정. 너무 많이 쓰면 과장되므로 제한
```

예시:

```text
沈泊川｜压低声音，紧张，呼吸发紧：
我刚才要是真推了门……

陆沉舟｜低声，稳，不急：
你会成为它最近的活人出口。
```

---

## 4. 긴 대사 처리

긴 대사는 한 번에 쓰지 말고 1~2문장 단위로 나눈다.

나쁜 방식:

```text
陆沉舟：开门是放它出来。动阵眼，才是让它松一点，又不至于冲出来。松少了没用，松多了失控。只给它留一口气。
```

좋은 방식:

```text
陆沉舟｜低声，稳，停半拍：
开门，是放它出来。

陆沉舟｜继续低声，语速慢：
动阵眼，才是让它松一点。

陆沉舟｜句尾压低：
松少了没用。

陆沉舟｜停半拍，声音更低：
松多了，失控。

陆沉舟｜最后一句压住：
只给它留一口气。
```

## 5. AI가 잘 알아듣는语气 단어

추천:

```text
低声
压着说
不喊
慢一点
短促
停半拍
声音发紧
气息不稳
尾音压低
像不想让别人听见
前半句快，后半句慢
```

주의:

```text
复杂
高级
宿命感
压迫感
电影感
强大气场
内心崩溃
```

위 단어들은 추상적이므로 단독으로 쓰지 말고, 반드시 보이는 행동이나 말투로 바꾼다.

예:

```text
不要只写：压迫感。
要写：低声，不喊，句子短，停半拍，视线不离开门框。
```

## 6. 주요 캐릭터별 대사 톤 카드

### 陆沉舟

```text
语气：低声，稳，压住情绪，不急，不喊
节奏：句子短，中间停半拍
尾音：压低，不上扬
感觉：像已经判断完结果，只是在通知别人
适合台词：警告、判断、规则说明、最终结论
```

예시:

```text
陆沉舟｜低声，稳，不喊，停半拍：
手别靠近。停住。
```

### 沈泊川

```text
语气：现实，紧张，呼吸发紧
节奏：前半句快，后半句慢下来
尾音：有一点不确定
感觉：嘴上还撑着，身体已经怕了
适合台词：追问、后怕、试探、现实风险判断
```

예시:

```text
沈泊川｜压低声音，紧张，呼吸发紧：
我刚才要是真推了门……
```

### 黑七

```text
语气：低声，警觉，带一点嘴硬
节奏：短句，反应快
感觉：不是搞笑，是害怕后还要嘴硬
适合台词：危险提醒、气味判断、嘴硬吐槽
```

예시:

```text
黑七｜低声，警觉，嘴硬但不轻浮：
现场谁不怕？不怕才假。
```

### 檀缺

```text
语气：冷静，轻，平
节奏：慢，不抢话
感觉：像观察完才补一句关键判断
适合台词：冷静补刀、事实判断、点破关键
```

예시:

```text
檀缺｜平静，轻声，不带情绪：
不是干净，是有人把痕迹抹掉了。
```

## 7. 영상 프롬프트에 넣을 때의 최종 형태

컷별 영상 프롬프트에서는 아래처럼 쓴다.

```text
【语音/台词执行】
陆沉舟｜低声，稳，不喊，句尾压低：
“手别靠近。停住。”

沈泊川｜紧张，呼吸发紧，前半句快后半句慢：
“我没碰。只是没想到，空宅里还能有动静。”
```

대사가 길면 다음처럼 같은 인물을 여러 줄로 나눠도 된다.

```text
陆沉舟｜低声，停半拍：
“开门，是放它出来。”

陆沉舟｜继续低声，慢一点：
“动阵眼，才是让它松一点。”
```

## 8. Gemini/ChatGPT 분진 작성 지침에 추가할 문장

```text
대사를 쓸 때는 AI 음성/립싱크가 자연스럽게 읽을 수 있도록 작성하라.
감정 설명은 대사 안에 넣지 말고, 대사 앞에 “语气 / 节奏 / 状态”로 분리하라.
긴 대사는 1~2문장 단위로 나누고, 필요한 곳에는 “……”로 호흡을 표시하라.
한 줄 대사는 실제 사람이 말할 수 있는 길이로 제한하라.
不要把演技说明写进台词里。
台词只保留角色真正说出口的话。
语气、停顿、呼吸、速度必须单独写在台词前。
```

## 9. 다음 작업 적용 규칙

다음에 영상 프롬프트를 쓸 때는 반드시 다음을 확인한다.

```text
1. 이 CUT의 주제와 감정이 무엇인지 먼저 확인한다.
2. 대사가 스토리 목적에 맞는지 확인한다.
3. 대사 앞에角色별语气/节奏/状态를 넣는다.
4. 긴 대사는 사람처럼 읽히도록 나눈다.
5. 대사 안에는 연기 설명을 넣지 않는다.
6. 지멍/Jimeng 영상 프롬프트에서는 화면 동작과语音提示를 분리한다.
```
