# 대사 읽기·영상 프롬프트 참조자료｜Voice Dialogue Video Prompt Reference

> 목적: 이후 사용자가 “영상용 프롬 써줘”, “대사 읽기 프롬 써줘”, “지멍 영상 프롬 써줘”라고 요청할 때, 대사가 AI 음성/립싱크에서 사람처럼 들리도록 참고하는 고정 기준이다.  
> 핵심: 대사는 실제 입으로 말할 문장만 쓰고, 감정·호흡·속도·시선·몸 상태는 대사 밖에 따로 쓴다.

> 우선순위: 이 문서는 대사를 새로 쓰는 규칙이 아니라, 확정된 원문 대사를 영상용 음성 지시로 정리하는 참조자료다. source index / canon / character / contract rules가 항상 우선한다.

---

## 1. 최우선 원칙

```text
대사 = 실제로 캐릭터가 말하는 문장만.
원문 대사 의미 = 반드시 보존.
연기/감정 = 대사 안에 섞지 말고 별도 지시로 분리.
호흡/멈춤 = 구두점과 짧은 지시로 표시.
한 줄 대사 = 실제 사람이 자연스럽게 말할 수 있는 길이.
```

영상 프롬프트 작성 시 대사는 문학식 서술이 아니라 **성우 지시서 + 원문 대사 보존** 구조로 작성한다.

원문 보존 규칙:

```text
대사 의미를 바꾸지 않는다.
source index의 확정 대사를 임의로 짧게 만들지 않는다.
초수 안에 말하기 어려우면 대사 삭제보다 CUT 분할, 초수 연장, 반응 컷, cutaway, 후시녹음/TTS 설계를 우선한다.
대사 문구 수정이 필요하면 원문 / 제안 / 수정 이유를 나란히 보여주고 사용자 확인을 받는다.
```

## 2. 나쁜 입력 방식

```text
陆沉舟冷冷地看着门框，眼神复杂又带着压迫感，声音低沉而危险地说：“手别靠近。停住。”
```

문제:

```text
1. 감정 설명이 대사 주변에 너무 많이 붙어 있다.
2. AI가 설명문과 대사를 혼동할 수 있다.
3. 음성 톤이 추상적으로 흐려진다.
```

## 3. 좋은 입력 방식

```text
角色：陆沉舟
语气：低声，压住情绪，不喊，慢半拍
状态：视线锁住铁门，身体不动
台词：手别靠近。停住。
```

장점:

```text
1. 누가 말하는지 명확하다.
2. 어떤 톤으로 말할지 명확하다.
3. 대사와 연기 지시가 분리되어 있다.
4. AI 음성/립싱크가 처리하기 쉽다.
```

## 4. 대사 길이 규칙

### 기본 기준

```text
원문 의미 보존 우선.
긴 설명 대사는 1~2문장 단위로 나눈다.
한 줄에 너무 많은 정보를 넣지 않는다.
5초 안에 긴 문장 2개 이상 금지.
10초 안에 3명 이상 장문 대사 금지.
```

### 나쁜 예

```text
陆沉舟：开门是放它出来。动阵眼，才是让它松一点，又不至于冲出来。松少了没用，松多了失控。只给它留一口气。
```

### 좋은 예

```text
陆沉舟｜低声，稳，不喊：
开门，是放它出来。

陆沉舟｜慢半拍，句尾压低：
动阵眼，才是让它松一点。

陆沉舟｜停半拍：
松少了没用。

陆沉舟｜更低：
松多了，失控。

陆沉舟｜压低尾音：
只给它留一口气。
```

## 5. 구두점 사용 기준

```text
， = 짧은 쉼.
。 = 확실한 멈춤.
…… = 망설임 / 숨 멈춤 / 말끝 삼킴 / 여운.
？ = 의심 / 반문 / 확인.
！ = 강한 감정. 과다 사용 금지.
```

예시:

```text
沈泊川｜紧张，呼吸发紧：
我刚才要是真推了门……

陆沉舟｜低声，稳，停半拍：
你会成为它最近的活人出口。
```

## 6. AI가 잘 이해하는 감정 지시어

### 좋은 지시어

```text
低声
压着说
不喊
慢一点
短促
停半拍
声音发紧
气息不稳
尾音压低
像不想让别人听见
嘴上撑着，身体已经怕了
```

### 피해야 할 추상어

```text
复杂
高级
宿命感
压迫感
电影感
强大气场
内心崩溃
```

추상어가 필요하면 반드시 보이는 행동으로 바꾼다.

```text
内心崩溃 → 呼吸变浅，眼神失焦，手指慢慢收紧。
压迫感 → 声音放低，句尾压住，身体不动，只用眼神盯住对方。
```

## 7. 캐릭터별 음성·대사 톤 카드

### 7.1 陆沉舟

```text
语气：低声，稳，压住情绪，不急，不喊。
节奏：必要时短，必要时说完整；判断前后留半拍。
尾音：压低，不上扬。
感觉：像已经读懂规则并正在带动现场，不是冷冰冰通知别人。
身体状态：身体不动，视线锁定目标，手部动作少而准。
```

예시:

```text
陆沉舟｜低声，稳，不喊，停半拍：
手别靠近。停住。
```

```text
陆沉舟｜低声，判断已定：
怕是对的。
```

---

### 7.2 沈泊川

```text
语气：现实，紧张，呼吸发紧。
节奏：前半句略快，后半句慢下来。
尾音：有一点不确定。
感觉：嘴上还撑着，身体已经怕了。
身体状态：眼神频繁看门、看手、看陆沉舟，后退时肩膀收紧。
```

예시:

```text
沈泊川｜压低声音，紧张，呼吸发紧：
我刚才要是真推了门……
```

```text
沈泊川｜故作镇定，尾音发虚：
这回我惜命。
```

---

### 7.3 黑七

```text
身份锁：小型黑色狐狸本体，不是人，不穿衣服，不狗化，不狼化。
语气：低声，警觉，带一点嘴硬。
节奏：短句，反应快。
感觉：不是搞笑，是害怕后还要嘴硬。
身体状态：耳朵竖起或压低，尾巴停住或轻摆，四足低位。
```

예시:

```text
黑七｜低声，警觉，嘴硬但不轻浮：
现场谁不怕？不怕才假。
```

```text
黑七｜耳朵猛地竖起，声音压低：
气松了。
```

---

### 7.4 檀缺

```text
语气：冷静，轻，平。
节奏：慢，不抢话。
尾音：干净收住。
感觉：像观察完才补一句关键判断。
身体状态：动作少，眼神从门框、阵痕、黑七之间轻移。
```

예시:

```text
檀缺｜平静，轻声，不带情绪：
不是干净，是有人把痕迹抹掉了。
```

```text
檀缺｜冷静，慢半拍：
阵根没动。你是故意留着。
```

## 8. 영상 프롬프트 안의 대사 작성 형식

### 상세형

```text
【语音提示】
角色：陆沉舟
语气：低声，稳，压住情绪，不喊
节奏：慢半拍，句尾压低
状态：视线锁住铁门，身体不动

【台词】
手别靠近。停住。
```

### 컷 안 간단형

```text
陆沉舟｜低声，稳，不喊：
手别靠近。停住。

沈泊川｜紧张，呼吸发紧：
我没碰。只是没想到，空宅里还能有动静。
```

영상용 프롬프트에서는 간단형을 우선 사용한다. 긴 대사이거나 감정이 중요한 컷은 상세형을 사용한다.

## 9. 분진 작성 시 추가해야 할 검사 항목

각 CUT의 대사 작성 후 반드시 아래를 확인한다.

```text
1. 대사와 연기 설명이 섞이지 않았는가?
2. 원문 대사 의미가 보존됐는가?
3. 초수 안에 말하기 가능한가?
4. 감정 지시가 추상어가 아니라 읽는 방식으로 되어 있는가?
5. 대사 사이에 필요한 쉼표, 마침표, 말줄임표가 있는가?
6. 캐릭터 톤이 맞는가?
7. 黑七가 인간화되지 않았는가?
8. 초수가 부족할 때 대사 삭제보다 분할/연장/반응 컷을 먼저 고려했는가?
```

## 10. Gemini/기타 모델에 추가할 지침

```text
대사를 쓸 때는 AI 음성/립싱크가 자연스럽게 읽을 수 있도록 작성하라.
감정 설명은 대사 안에 넣지 말고, 대사 앞에 “语气 / 节奏 / 状态”로 분리하라.
긴 대사는 1~2문장 단위로 나누고, 필요한 곳에는 “……”로 호흡을 표시하라.
한 줄 대사는 실제 사람이 말할 수 있는 길이로 제한하라.
원문 대사 의미를 임의로 줄이거나 바꾸지 마라.
초수가 부족하면 먼저 CUT 분할, 초수 연장, 반응 컷, cutaway를 제안하라.
대사 문구 수정이 필요하면 원문 / 제안 / 수정 이유를 나란히 보여줘라.
不要把演技说明写进台词里。
台词只保留角色真正说出口的话。
语气、停顿、呼吸、速度必须单独写在台词前。
```

## 11. 초수 대비 대사량 검사 규칙

```text
각 CUT의 초수와 대사량을 반드시 맞춰라.
중국어 대사는 자연스러운 연기 기준으로 1초당 약 3~4자 정도로 본다.
빠른 말투가 필요한 경우에도 1초당 5자를 넘기지 않는 것을 권장한다.
초수 안에 말하기 어려우면 먼저 CUT을 늘리거나, 대사를 여러 speaking node로 나누거나, 반응 컷/cutaway를 넣어라.
원문 의미를 줄이는 대사 삭제/축약은 사용자 승인 전까지 금지한다.
5초 안에 대사가 15~20자를 넘으면 위험하다.
10초 안에 3명 이상이 연속으로 긴 문장을 말하면 위험하다.
자체 점검에서 “문제 없음”이라고 쓰지 말고, 초수 대비 대사량이 과한 CUT을 반드시 찾아서 지적하라.
```

## 12. 영상 프롬프트 작성 시 적용 규칙

사용자가 “영상용 프롬 써줘”라고 하면 다음을 적용한다.

```text
1. 컷의 주제와 스토리 기능을 먼저 확인한다.
2. 대사 톤은 캐릭터별 음성·대사 톤 카드에 맞춘다.
3. 대사는 실제 말할 문장만 적고 원문 의미를 보존한다.
4. 감정·호흡·속도·시선은 대사 앞에 별도 표기한다.
5. 과밀 대사는 나누거나, 컷 길이를 조정하도록 사용자에게 알린다. 대사 축약은 사용자 승인 후에만 한다.
6. 지멍/Jimeng 영상 프롬프트에는 너무 긴 음성 지시를 넣지 않고 핵심만 남긴다.
7. 립싱크가 중요한 컷에서는 한 컷 한 화자 또는 짧은 교대 대사로 설계한다.
8. 감정이 중요한 컷에서는 “语气 / 节奏 / 状态 / 台词” 구조를 사용한다.
```

## 13. 최종 한 줄 기준

```text
AI가 사람처럼 말하게 하려면, 문학 문장이 아니라 성우 지시서 + 짧은 대사로 입력한다.
```
